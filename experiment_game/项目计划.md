# 实验诱导与脑电接收模块 · 项目计划

> **版本**：v0.1  
> **日期**：2026-07-22  
> **状态**：仅计划文档，本阶段不生成业务代码  
> **本目录职责**：第一人称运动想象实验的**视觉诱导** + **OpenBCI 数据接收/打标/落盘**  
> **范式唯一标准**：`project_mi/资料/实验诱导游戏说明/基于第一人称虚拟身体交互的渐进式运动想象脑机接口训练与数据采集网页游戏系统(1).md`

---

## 0. 一句话目标

搭建一套可跑通的实验系统：受试者在网页第一人称场景中按标准试次做左右手抓取运动想象；本机用 OpenBCI Cyton **BrainFlow 直连**采 EEG，并在**同一 LSL 时钟**上写入试次事件，落盘后可供 `project_mi` 预处理与离线模型使用。

## 1. 与仓库其他部分的边界

| 目录 / 模块 | 做什么 | 本项目怎么用 |
|-------------|--------|--------------|
| **本目录 `experiment_game/`** | 诱导前端、实验状态机、Marker、会话录制 | **正式实验主路径** |
| `project_mi/资料/实验诱导游戏说明/...` | 范式设计说明书 | **需求与时序唯一依据** |
| `project_mi/code/preprocess_lab` | 切 4s 窗、标签 `y_task` / `y_three` | 本项目产出需能被其（或后续适配器）吃掉 |
| `project_mi/code/train_lab` | EEGNet 等离线训练 | 用正式采集数据；不定义实验范式 |
| `collect_data/.../lsl_connect` | BrainFlow→LSL、录制、控制台 | **复用采集能力**，尽量薄封装，不推倒重写 |
| `collect_data/.../MI_model` | 连续车辆想象 + CSP/SVM | **另一数据集上的离线基线**，与本范式无关 |

**明确不做（本阶段）：**

- 不以 OpenBCI 官方 GUI 作为数据采集中枢（GUI 仅可选作波形监视）。
- 不在本目录重做一套 Cyton 底层驱动。
- 不把 BCI2a / 车辆连续想象范式写进本实验流程。
- 本阶段不写实现代码（本文档之后按阶段再开）。

## 2. 技术栈选型（已定）

| 层级 | 选型 | 说明 |
|------|------|------|
| 设备读数 | Python + **BrainFlow** 直连 Cyton | 最低合理延迟；复用 `lsl_connect` |
| 流协议 | 本进程发布 **LSL**：`OpenBCI_EEG` + `OpenBCI_Markers` | 本机进程间；科研工具可订阅 |
| 实验控制 | Python 实验控制器（状态机 + 打标 + 录制） | Cue/MI/Rest 时间以控制器为准 |
| 诱导前端 | 浏览器 + **Three.js**（MVP） | 符合「网页游戏」定位，迭代快 |
| 前后端桥 | **WebSocket**（本机） | 浏览器不直连串口；事件上报给 Python 打 LSL 时戳 |
| 落盘 MVP | `eeg.csv` + `events.jsonl` + `session.meta.json` | 进阶可选 LabRecorder / XDF |
| 操作员面 | 轻量控制页或沿用/精简现有控制台 | 串口、开始/停止会话、是否 synthetic |
| 时钟 | 事件时间戳用 `pylsl.local_clock()` | 与 EEG 流对齐，禁止只用墙钟当主轴 |

**不采用作为主路径：**

- OpenBCI GUI → Networking Widget → LSL → 本程序订阅（多一跳、缓冲与时间戳风险更高）。
- 从零实现 Cyton 串口协议。

**延迟预期（定性）：**

- BrainFlow 直连 + 本机 LSL：对**离线试次对齐**足够；端到 LSL 目标约几十毫秒量级（与现有 `collect_data` 文档一致）。
- 当前阶段目标是**训练/采数对齐**，不是亚帧级在线闭环；在线反馈留到后期阶段评估。

## 3. 实验范式摘要（摘自标准文档，实现必须遵守）

### 3.1 三类标签（数值约定与 `y_three` 对齐）

| 数值 | 名称 | 含义 |
|------|------|------|
| 0 | Rest | 静息 |
| 1 | Left | 左手抓取运动想象 |
| 2 | Right | 右手抓取运动想象 |

展示文案可用 `Left MI` / `Right MI` / `Rest`，落盘与训练以数值为准。

### 3.2 三大实验阶段

| 阶段 | 名称 | 脑电 | 视觉要点 |
|------|------|------|----------|
| 一 | 环境适应 Environment Adaptation | **不采正式数据**（可不录或单独标记 `phase=adapt`） | **引导式短流程**；熟悉双手、物品、任务规则；约 3–5 分钟 |
| 二 | 运动想象学习 Motor Imagery Learning | 默认可录但 **`phase=learn`，不进主训练集** | Step1 完整抓取 → Step2 **前伸无抓握** → Step3 无动画自主想象 |
| 三 | 正式采集 EEG Data Acquisition | **主数据集** | **无手部动画**；双手**静态可见** + 静态目标 + 标准提示 |

准入：文档要求 ERD / 主观确认 / 简易分类可分。实现上 **MVP 用人工确认 + 可选短测 session**；实时 ERD 门控后置。

### 3.3 正式采集单 Trial 时序（标准 14s 结构）

以范式文档为准，实现时状态机按下列段驱动，并打 Marker：

| 相对时间 | 时长 | 阶段名 | 画面 | 是否进训练窗 |
|----------|------|--------|------|----------------|
| 0–2 s | 2 s | Fixation | 十字注视；可有轻微聚焦引导 | 否（或仅作基线参考，切窗规则另定） |
| 2–4 s | 2 s | Cue | 目标物品 + 左/右文字/语音提示 | **Cue 起点 = `cue_onset`**（对齐预处理「Cue 后 0–4 s」） |
| 4–8 s | 4 s | MI | **完全静画**，无手动画 | **是**：Left/Right 分类窗 |
| 8–9 s | 1 s | PostMI_hold | 画面保持、无突变 | 否（Discard） |
| 9–13 s | 4 s | Rest | **完全静止** | **是**：Rest 窗 |
| 13–16 s* | 3 s | Transition | 视角缓慢下一目标 | 否（Discard） |

\* 文档将休息缓冲写为「总 8s = 1 + 4 + 3」。**已冻结：单 trial 状态机含 Transition 3 s（总计约 17 s）**，对外说明与范式文档「8s 缓冲」对应。游戏画面细则见 `docs/游戏内容规格.md`。

**硬性禁区（正式采集）：**

- MI 4 s：禁止手/物/镜头运动；双手可显示但必须静止。
- Rest 4 s：禁止视角漫游；漫游只允许在 Transition。

### 3.4 任务与轮换规则

- 唯一动作内核：手部**抓取**想象（不设挥手、推拉等）。
- 同物品双向配对：1 物品 → 左+右各 1 trial；左右顺序**组内随机**。
- 每 2 trial 换物品；**组间漫游不另开计时段**，并入该 trial 末 3 s Transition（换物时加大幅度，标 Transition/Discard）。
- 每 10 trial 换大场景（家庭 / 医院 / 学校 / 超市 / 公园）；**MVP：家庭桌面 + 杯子，不换景。**

### 3.5 与离线切窗的对齐原则

现有 `preprocess_lab`：`cue` 后 0–4 s → MI 样本；Rest 为独立 4 s 窗。

本系统必须保证：

1. 每个正式 trial 有明确的 `cue` 事件（带 `label∈{1,2}`）。
2. `mi_start` / `mi_end` 与 Cue 后 4 s 一致（或由 `cue` + 固定偏移推导）。
3. `rest_start` / `rest_end` 覆盖静止 4 s，**不含** Transition。
4. 学习阶段动画 trial 不得与正式 trial 混进同一训练清单（靠 `phase` 过滤）。

## 4. 系统架构（逻辑）

```text
                    ┌─────────────────────────────────────┐
                    │  浏览器诱导 (Three.js)                 │
                    │  适应 / 学习 / 正式试次画面与提示      │
                    └─────────────────┬───────────────────┘
                                      │ WebSocket
                                      │ (trial 阶段到达、就绪、人工标记等)
                                      ▼
┌──────────────┐   控制    ┌──────────────────────────────────┐
│ 操作员界面    │─────────►│  实验控制器 ExperimentController   │
│ 串口/开始停止 │          │  · 试次状态机（定时）               │
└──────────────┘          │  · 推 LSL Markers (local_clock)    │
                          │  · 写 events.jsonl                 │
                          │  · 启停录制                        │
                          └─────────────┬────────────────────┘
                                        │ 复用 / 调用
                          ┌─────────────▼────────────────────┐
                          │  采集服务 (BrainFlow → LSL EEG)    │
                          │  复用 collect_data/lsl_connect     │
                          └─────────────┬────────────────────┘
                                        │ LSL: OpenBCI_EEG
                                        ▼
                          ┌─────────────────────────────────┐
                          │  Recorder：eeg.csv + meta.json   │
                          │  （可选同时 LabRecorder → XDF）   │
                          └─────────────────────────────────┘
```

**原则：** 画面可以在浏览器按本地计时切换，但**权威时间轴在 Python 控制器**；前端「阶段到达」仅作确认或 UI 同步，打标以控制器 `local_clock()` 为准（避免浏览器卡顿导致标签漂移）。推荐模式：**控制器主导倒计时，前端只渲染当前阶段**。

## 5. 数据与事件约定（计划级）

### 5.1 会话目录结构（计划）

```text
data/sessions/
  {subject_id}_{session_id}_{YYYYMMDD_HHMMSS}/
    session.meta.json      # 被试、阶段、通道、采样率、软硬件版本
    eeg.csv                # lsl_time + 8 通道（与现有采集列名对齐）
    events.jsonl           # 一行一个事件
    trial_table.csv        # 可选：每 trial 一行汇总（由 events 生成）
```

### 5.2 通道顺序（与现有采集对齐）

```text
C3, C4, CZ, CP3, CP4, CPZ, FC3, FC4
```

采样率：**250 Hz**。质控分析可强调 C3/C4，存盘必须 8 通道。

### 5.3 Marker / 事件类型（草案）

| event | 何时 | 必要字段 |
|-------|------|----------|
| `session_start` / `session_end` | 会话起止 | subject, session, phase 模式 |
| `phase_start` / `phase_end` | adapt / learn / acquire | `phase` |
| `trial_start` / `trial_end` | 每 trial | `trial_id`, `object`, `scene` |
| `fixation` | 注视开始 | `trial_id` |
| `cue` | 目标+左右提示出现 | `trial_id`, `label`∈{1,2} |
| `mi_start` / `mi_end` | MI 窗 | `trial_id`, `label` |
| `rest_start` / `rest_end` | 静止 Rest 窗 | `trial_id`, `label=0` |
| `transition` | 视角趋近/组间漫游 | `trial_id`（可空） |
| `object_change` / `scene_change` | 换物/换景 | 新 object / scene |
| `trial_reject` | 操作员或自动质控标记无效 | `reason` |

每条事件至少含：`t_lsl`, `event`, 以及上表字段。

### 5.4 LSL 流

| 流名 | 类型 | 说明 |
|------|------|------|
| `OpenBCI_EEG` | EEG | 8 ch @ 250 Hz（采集服务发布） |
| `OpenBCI_Markers` | Markers | 字符串或编码事件（控制器发布） |

## 6. 端到端业务流程（操作员视角）

```text
1. 连接 Cyton（或开启 synthetic 板联调）
2. 启动采集服务 → 确认 LSL EEG 有数据
3. 录入 subject_id / session 元信息
4. 打开诱导页，进入【适应】→ 不进主数据集
5. 进入【学习】Step1→2→3 → phase=learn
6. 【准入】人工确认（MVP）或跑短测
7. 进入【正式采集】
   - 自动：fixation → cue → mi → hold → rest → transition
   - 每 2 trial 换物；每 10 trial 换景（完整版）
   - 全程写 eeg.csv + events.jsonl
8. 结束会话 → 质检摘要（丢包、断流、reject 数）
9. 导出 / 交给预处理：按 cue/rest 切 4s 窗 → y_three
10. 离线训练（Rest vs Task → Left vs Right 等，由 project_mi 负责）
```

## 10. 风险与对策

| 风险 | 对策 |
|------|------|
| 浏览器卡顿导致阶段漂移 | **控制器主导计时**，前端只显示 |
| GUI 与 BrainFlow 抢 COM | 文档与启动检查禁止 GUI 串口直播；监视用 UDP/LSL 旁路 |

| 与旧 `MI_model` 文件名标签习惯混淆 | 文档与目录隔离；本项目只用 events 标签 |
| 实时 ERD 准入过早承诺 | MVP 人工准入；算法准入后置 |
