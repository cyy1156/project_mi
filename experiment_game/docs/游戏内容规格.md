# 游戏内容规格（诱导前端）

> **版本**：v0.1  
> **日期**：2026-07-22  
> **状态**：内容定稿（实现前可再改，改动需同步 `项目计划.md`）  
> **范式依据**：`project_mi/资料/实验诱导游戏说明/基于第一人称虚拟身体交互的渐进式运动想象脑机接口训练与数据采集网页游戏系统(1).md`

本文只约定**受试者看到/经历什么**，不涉及采集与落盘实现。

---

## 1. 产品定位

第一人称抓取运动想象**诱导环境**（非闯关得分游戏）：

- 唯一动作内核：想象用手**抓取**目标物（无挥手、推拉、旋转等）
- 三类状态：Left MI / Right MI / Rest
- 目标：先建立体感，正式采集时避免视觉运动伪迹污染脑电

---

## 2. 已冻结决策

| 项 | 定稿 |
|----|------|
| 正式 MI 双手 | **静态可见**（无动画、无位移）；不隐藏 |
| 学习 Step 2 | 手**往前伸**，**无抓握闭合** |
| Transition | **每个 trial 末固定 3 s**，移向下一 trial 目标 |
| 组间漫游 | **不另开计时段**；换物时在同段 Transition 内加大幅度/路径 |
| 适应阶段 | **引导式短流程**（非自由闲逛） |
| MVP 场景与物 | **家庭桌面 + 杯子**（Phase 2）；Phase 3 起正式段支持多物多景 |
| 演示取杯（Step1 / 适应示意） | 完整抓取后**将杯子取走**（抬起移出桌面）；下一 trial 开始前杯子复位 |
| 操作者 / 被试按键 | **Phase 3 已实现**，见 `docs/操作按键设计.md` |
| 换物 | 正式采集每 **2 trial** 换物（cup → bottle → apple 循环） |
| 换景 | 正式采集每 **10 trial** 换景（home / hospital / school） |

---

## 3. 画面常驻规则

| 元素 | 规则 |
|------|------|
| 视角 | 始终第一人称；禁止第三人称、旁观角色 |
| 虚拟双手 | 全程显示；位置/高度/间距贴近真人习惯 |
| 目标物 | 置于双手可自然够到的范围 |
| 场景 | 只换背景与物品池，不改「抓取」任务内核 |

**正式采集 MI / Rest：** 双手可见但**完全静止**；物品与镜头亦静止。

---

## 4. 三大阶段

### 4.1 阶段一 · 环境适应（引导式，约 3–5 分钟）

- **目的**：熟悉双手、桌面、杯子与任务规则；统一初始状态  
- **脑电**：不进主数据集（可不录或 `phase=adapt`）  
- **流程要点**（引导短流程，非自由探索）：
  1. 进入家庭桌面第一人称视角，看到双手与杯子  
  2. 简短说明：任务是想象左/右手去抓杯子  
  3. 可选 1～2 次示意（操作员/系统推进），确认受试者理解后结束  

具体步数与文案实现时可再细化，但不得改成无引导的自由漫游。

### 4.2 阶段二 · 运动想象学习

- **脑电**：默认可录，`phase=learn`，**不进主训练集**  
- **Step 1 完整观察（演示）**：伸手 → 手指闭合抓取 → **抬起并取走杯子** → 手复位；杯子在下一 trial 前放回桌面；受试者只观察  
- **Step 2 弱辅助**：目标手**前伸靠近物体**，**不做抓握、不取走杯子**；受试者边看边自主想象抓取  
- **Step 3 无辅助**：与正式视觉接近——静物 + 静双手 + 文字/语音；纯自主想象  

**演示取杯细则（Step1 / 适应示意共用）：**

| 子段（相对 MI 窗） | 画面 |
|--------------------|------|
| 前约 30% | 手伸向杯子 |
| 约 30%–50% | 手指闭合抓住 |
| 约 50%–75% | 抬起杯子并向身旁/身前移出桌面（取走） |
| 约 75%–100% | 空手复位；杯子保持在画面外或桌下不可见 |
| 下一 trial / 阶段切换 | 杯子瞬移或淡入回桌面原位 |

### 4.3 准入

- 范式：ERD 可分 + 主观确认 + 简易分类可分  
- **MVP**：人工确认（+ 可选短测）；实时 ERD 门控后置  

### 4.4 阶段三 · 正式采集

- **无任何手部抓取动画**；双手静态可见  
- MI 4 s、Rest 4 s：禁止手/物/镜头运动  
- 漫游仅允许在 Transition（见 §5、§6）

---

## 5. 正式单 Trial 画面时间线

单 trial 状态机**含**末尾 Transition，墙钟约 **17 s**（对外可称核心 14 s + 休息缓冲 8 s = 1+4+3）。

| 阶段 | 时长 | 画面 | 受试者 |
|------|------|------|--------|
| Fixation | 2 s | 居中十字；可缓慢聚焦引导至目标 | 注视、平复 |
| Cue | 2 s | 突出杯子（或当前目标）+ 左/右文字与语音 | 明确本 trial 用手 |
| MI | 4 s | **完全静画**（静双手 + 静物） | 只在脑中想象抓取 |
| PostMI_hold | 1 s | 无突变保持 | 收束 |
| Rest | 4 s | 视角锁定、全静 | 放松静息 |
| Transition | 3 s | 视角缓慢移向**下一 trial** 目标 | 跟看；**不进 Rest 窗** |

---

## 6. 换物、Transition 与「组间漫游」合并规则

### 6.1 物品与左右手

- 同物品双向配对：1 物 → 左 1 + 右 1 trial；组内左右顺序随机  
- 每完成 2 trial → 换新目标物（完整版）；MVP 可长期只用杯子，仍保留配对与随机左右  

### 6.2 Transition（每 trial 必有，兼组间漫游）

```text
每个 trial 末：固定 3 s Transition → 下一 trial 的目标位置

下一 trial 仍是同一物品（组内第 2 个）：
  → 小幅挪近 / 微调视角

下一 trial 是新物品（刚完成一对，换物）：
  → 同一 3 s 内幅度略大 / 路径稍长
    （即范式中的「组间视角漫游」，不另开计时段）
```

- Rest 4 s **永不**含漫游  
- MVP：**不**在 Transition 之外再插独立「组间漫游」阶段  

### 6.3 换大场景（完整版）

每 10 trial 换景：家庭 / 医院 / 学校 / 超市 / 公园。  
**MVP：固定家庭桌面，不换景。**

---

## 7. MVP 范围裁切

| 内容 | MVP | 完整版 |
|------|-----|--------|
| 场景 | 家庭桌面 | 五场景轮换 |
| 物品 | 杯子（可后扩少量物） | 多物品池 + 双 trial 换物 |
| 双手 | 第一人称；正式段静态可见 | 同左 |
| 学习 | Step1 + Step2（前伸无抓）+ Step3 | 同左，节奏可加长 |
| 漫游 | 仅 trial 末 3 s Transition（换物时加幅） | 同左；可选再增强 |
| 准入 | 人工确认 | ERD / 短测 |

---

## 8. 一局叙事顺序

```text
录入信息 / 告知
  → 适应：引导短流程（家庭桌面 + 杯子）
  → 学习：Step1 → Step2（前伸无抓）→ Step3
  → 准入：人工确认（MVP）
  → 正式：注视 → 提示 → 静想象 → 保持 → 静休息 → Transition
        （每 2 trial 换物时 Transition 加幅；MVP 可暂不换物）
  → 结束
```

---

## 9. 操作者 / 被试按键（Phase 3 已实现）

双角色分工，避免受试者误触实验控制：

| 角色 | 职责概要 |
|------|----------|
| **被试（Subject）** | 仅确认「已阅读 / 继续」类提示；不控制试次启停与 reject |
| **操作者（Operator）** | 暂停、代确认、准入、reject、紧急中止 |

完整键位见：[`docs/操作按键设计.md`](./操作按键设计.md)。
