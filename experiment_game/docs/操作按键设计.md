# 操作者 / 被试按键设计

> **版本**：v0.2  
> **日期**：2026-07-23  
> **状态**：**Phase 3 已实现**  
> **配套**：`游戏内容规格.md`、`ws_protocol.md`、`项目计划.md`

---

## 1. 目标

实验现场通常有两人：

- **被试（Subject）**：做运动想象，注意力在画面上  
- **操作者（Operator）**：控流程、看信号、处理异常  

需要**分开的按键/控件**，避免被试误触「结束会话 / reject / 跳过」等危险操作。

---

## 2. 角色与权限

| 能力 | 被试 | 操作者 |
|------|------|--------|
| 确认说明弹窗（继续 / 我明白了） | ✓ | ✓（可代点） |
| 学习各 Step「开始本步」 | ✓ | ✓ |
| 正式采集准入确认 | ✗ | ✓（`allow_subject=false`） |
| 暂停 / 继续计时 | ✗ | ✓ |
| 标记当前 trial `reject` | ✗ | ✓ |
| 紧急停止采集 | ✗ | ✓ |

---

## 3. 键位（已实现）

### 3.1 被试端

| 按键 | 动作 |
|------|------|
| `Space` / `Enter` | 确认当前 prompt（准入弹窗除外） |

### 3.2 操作者端（页顶操作者条 + 键盘）

| 按键 / 按钮 | 动作 |
|-------------|------|
| `P` / 暂停 | 暂停 / 恢复（冻结 trial 时钟） |
| `N` / 代确认 | 代被试确认 prompt |
| `G` | 准入确认（`gate_ok`） |
| `R` / Reject | 标记当前 trial 为 reject（写入 `trial_reject`） |
| `Esc` / 中止 | 紧急结束会话（二次确认） |
