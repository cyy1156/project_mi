# MI 运动想象分类 · 离线训练项目计划

> **版本**：v1.4  
> **范围**：仅离线训练与验证；**基线模型 CSP + SVM**  
> **策略**：从 **二分类** 逐步扩容至 **五分类**，每阶段重复同一套传统训练流程  
> **数据根目录**：`MI_model/data/`（本项目 `RecorderWorker` 采集的 CSV）  
> **采集范式**：**纯想象**——每个 CSV 为一次连续想象录制，全程无注视/箭头/休息分段

---

## 1. 项目目标

在固定 8 通道、纯运动想象范式下，按版本递增类别数，逐步建立并验证 **CSP + SVM** 基线分类器：

```text
CSV 连续想象段（采集端已滤波）
  → 数据检查
  → 去 warmup + MI 窄带通(8~30Hz) + 滑窗切样本
  → X.npy / y.npy / groups.npy
  → 按 group 划分 80% 训练 / 20% 测试（固定）
  → CSP + SVM（仅训练集 fit + 训练集内 5 折 CV）
  → 测试集评估（步骤 4，模型未见过）
  → 导出模型（csp.pkl + svm.pkl）
```

---

## 2. 版本路线（先前后二分类 → 逐步五分类）

当前数据质量下，**唯一可靠的首个二分类为前进 / 后退**；左右、部分 stop/front 等 session 已用文件名 **`_no`** 标记为不可用。

| 阶段 | 类别 | 标签 | 使用 CSV（不含 `*_no.csv`） | 进入下一阶段条件 |
|------|------|------|------------------------------|------------------|
| **Stage 1** | 二分类 | `front=0`, `back=1` | `front*.csv`, `back*.csv` | 5 折 CV **≥ 70%** |
| **Stage 2** | 三分类 | + `stop=2` | Stage 1 + `stop*.csv`（无 `_no`） | 同上 |
| **Stage 3** | 四分类 | + `left=3` | Stage 2 + `left*.csv`（重录后去掉 `_no`） | 同上 |
| **Stage 4** | 五分类 | + `right=4` | Stage 3 + `right*.csv`（重录后去掉 `_no`） | 同上 |

**Stage 4 最终标签**：

| 标签 | 想象内容 | 文件前缀 |
|------|----------|----------|
| 0 | 前进 | `front` |
| 1 | 后退 | `back` |
| 2 | 停止 | `stop` |
| 3 | 左转 | `left` |
| 4 | 右转 | `right` |

**文件命名约定**：

| 模式 | 示例 | 行为 |
|------|------|------|
| 正常 session | `back.csv`, `front1.csv` | 按前缀匹配类别，可参与训练 |
| 质量标记 **`_no`** | `front_no.csv`, `right1_no.csv` | **程序自动排除**，永不进入数据集 |
| 同类别多次录制 | `back.csv`, `back1.csv` | 标签相同，各算 1 个 session |

### 3.1 文件位置与格式

```text
MI_model/data/C3  C4  CZ  CP3  CP4  CPZ  FC3  FC4/
├── front1.csv              # Stage 1 可用：纯想象前进

├── left.csv / stop1.csv    # 待检或后续 Stage 使用
└── …
```

**CSV 表头**：

```text
lsl_time, C3, C4, CZ, CP3, CP4, CPZ, FC3, FC4
```

- 单位：**µV**（ADC 缩放后写入 LSL）
- 采样率：250 Hz
- 通道数：8
- **采集端预处理**：已在 `config/default.yaml` → `AcquisitionWorker` → `lsl_connect/preprocessing.py` 完成（见 §3.5）

### 3.2 当前数据清单（2026-06 更新）

每个 CSV = **一段连续纯想象**，时长约 **82~87 s**。

**Stage 1 参与训练（无 `_no` 且属 front/back）**：

| 类别 | 文件 | 说明 |
|------|------|------|
| front (0) | `front1.csv` | 1 个 session |
| back (1) | `back.csv` | **仅 1 个 session**（与 front 对齐；`back1.csv` 暂不参与） |

**session 数量对齐**：各类可用 CSV 数取最小值（`min(front, back, …)`），按文件名排序取前 N 个；配置见 `BALANCE_SESSIONS_PER_CLASS`。

**已标记 `_no`（自动排除）**：

`front_no.csv`, `left1_no.csv`, `right_no.csv`, `right1_no.csv`, `stop_no.csv`

**其余文件**：留待 inspect 或后续 Stage（如 `left.csv`、`stop1.csv`）；重录左右类时勿加 `_no` 后缀。

### 3.3 滑窗样本估算（Stage 1：front / back）

| 项目 | 数值 |
|------|------|
| Stage 1 session 数 | **3**（front1 + back + back1） |
| 每 session 约 | **158 窗** |
| 合计约 | **≈ 475 样本** |
| 分组（4 s epoch） | 约 **60 组**（够 5 折 GroupKFold） |

front 仅 1 个 session，类别略不平衡；补录 `front.csv` 或更多 `front*.csv` 可改善。

### 3.4 数据质量注意

1. **`_no` 后缀优先**：人工标记为质差的数据，代码层直接跳过，不依赖幅值阈值。
2. inspect 仍会对非 `_no` 文件做幅值检查；超阈值写入 `skip_files`（与 `_no` 独立）。
3. 左右等类全段幅值异常的文件应 **重录** 或保持 `*_no.csv` 命名，待 Stage 3/4 再纳入。
4. CSV **无 marker 列**；类别由 **文件名前缀** 决定。

### 3.5 预处理状态（采集端已完成）

CSV 中的数值 **不是 raw ADC**，而是经主工程采集链路处理后的 µV：

```text
BrainFlow 原始计数
  → counts_to_microvolts（缩放为 µV）
  → 带通 0.5~45 Hz（Butterworth zero-phase）
  → 陷波 49~51 Hz（50 Hz 工频）
  → LSL 广播 → RecorderWorker 写入 CSV
```

对应 `config/default.yaml`：

```yaml
滤波:
  启用: true
  带通低频_hz: 0.5
  带通高频_hz: 45.0
  陷波低频_hz: 49.0
  陷波高频_hz: 51.0
```

**离线训练管道因此：**

| 步骤 | 离线是否再做 | 说明 |
|------|:------------:|------|
| ADC → µV 缩放 | **否** | CSV 已是 µV |
| 0.5~45 Hz 带通 | **否** | 采集端已做，勿重复 |
| 49~51 Hz 陷波 | **否** | 采集端已做，勿重复 |
| **8~30 Hz 带通** | **是** | 在已滤波数据上 **窄化** 至 MI μ/β 频段 |
| warmup 丢弃 | **是** | 去除录制开头伪影 |

> 二次宽带走带通或陷波会改变频响、引入额外相位处理，**禁止**在 `02_build_dataset.py` 中重复。

## 4. 采集范式（纯想象）

### 4.1 与旧方案的区别

| 项目 | 旧假设（已废弃） | **当前实际范式** |
|------|------------------|------------------|
| 结构 | 9 s/trial：注视→箭头→想象→休息 | **无分段**，全程想象 |
| CSV 含义 | 多 trial 连续拼接 | **1 个 CSV = 1 次 session = 1 类想象** |
| 有效 EEG | 仅 trial 内 4 s 想象段 | **整段（去掉 warmup）均可切窗** |
| 标签来源 | trial 级事件标记 | **文件名前缀**（`back.csv` = 后退） |

### 4.2 单次录制流程

```text
开始采集 + 开始录制 → 立即进入「纯想象 front」→ 持续 ~80 s → 停止录制
保存为 front1.csv（质差则改名为 front_no.csv 排除）
```

各类别想象内容（统一「坐在车内」）：

| 文件前缀 | 被试任务 | Stage 1 |
|----------|----------|:-------:|
| front | 持续想象 **前进** | ✓ |
| back | 持续想象 **后退** | ✓ |
| left | 持续想象 **向左转向** | 后续 Stage 3 |
| right | 持续想象 **向右转向** | 后续 Stage 4 |
| stop | 持续想象 **停车 / 制动** | 后续 Stage 2 |

### 4.3 电极（CH1~CH8）

C3 · C4 · CZ · CP3 · CP4 · CPZ · FC3 · FC4（以 CSV 列顺序为准）

### 4.4 补采建议

| 项目 | 建议 |
|------|------|
| 单次 session 时长 | **60~120 s**（当前 ~83 s 合理） |
| 每类 session 数 | 当前 2 次 → 目标 **≥ 10 次/类** |
| 文件命名 | `left.csv`、`left1.csv`… 或带时间戳 `left_20260603.csv` |
| 间隔 | 类间休息 ≥ 30 s，换类前确认状态恢复 |

## 5. 关键参数（写入 `config.py`）

| 项目 | 参数 | 说明 |
|------|------|------|
| 采样率 | `SFREQ = 250` | Hz |
| 采集端滤波（已完成） | 0.5~45 Hz 带通 + 49~51 Hz 陷波 | 见 `default.yaml`，离线 **不重复** |
| 离线 MI 窄带通 | `8 ~ 30` Hz | 唯一离线滤波步骤，缩窄至 μ/β |
| warmup 丢弃 | **2 s**（500 点） | 去除录制开头伪影/稳定期 |
| 滑窗 | 窗长 **2 s**（500 点），步长 **0.5 s**（125 点） | 在整段可用数据上滑动 |
| CV 分组 epoch | **4 s**（1000 点） | 同一 4 s 块内滑窗样本共享 group_id，防泄漏 |
| CSP 输出维度 | `4` | |
| 分类器 | `SVC(kernel='rbf', C=1.0)` | |
| 训练 / 测试划分 | **80% / 20%** | `GroupShuffleSplit`，按 **group** 留出，`SPLIT_RANDOM_STATE=42` |
| 交叉验证 | **5 折 GroupKFold** | 仅在 **训练集** 内；`groups` = 4 s epoch |
| 样本形状 | `X: (N, 8, 500)`, `y: (N,)`, `groups: (N,)` | 另存 `train_idx.npy`, `test_idx.npy` |

### 5.1 样本与分组逻辑

```text
1 个 CSV（session，µV，采集端已 0.5~45Hz+陷波）
  → 去掉前 2 s warmup
  → 仅做 8~30 Hz 带通（不再陷波、不再 0.5~45Hz）
  → 按 4 s 划 epoch（epoch_id = 0, 1, 2, …）
  → 在每个 epoch 内（及跨 epoch 边界按窗中心所属 epoch 归属）
     2 s 窗 / 0.5 s 步长切样本
  → group_id = f"{session_name}_ep{epoch_id}"
  → label = 文件名前缀（front=0, back=1, …）；`*_no.csv` 跳过
```

**为何用 4 s epoch 分组**：纯想象只有 2 个 session/类，若按 session 分组仅 4 组，无法做 5 折 CV；按 4 s epoch 分组可在保持「相邻高度相关样本不跨折」的同时凑够分组数。

## 7. 传统训练步骤（每个 Stage 重复执行）

### 步骤 1：数据检查 — `01_inspect_data.py`

**输入**：当前 Stage 涉及的 CSV  
**输出**：`reports/stage{N}/inspect_report.txt`

| 检查项 | 说明 |
|--------|------|
| 文件 ↔ 标签 | 确认文件名前缀与 Stage 类别表一致 |
| session 时长 | 每个 CSV 行数、秒数 |
| warmup 段质量 | 前 2 s 幅值是否异常，是否需加长 discard |
| 全段统计 | 各通道 mean/std/max |
| 异常 session | 整段幅值异常则 flag 整文件 skip |
| 类别平衡 | 各类 session 数、总时长 |

**步骤验收**：报告列出可用 session 清单与需 skip 文件。

---

### 步骤 2：预处理 + 数据集生成 — `02_build_dataset.py`

**输入**：步骤 1 确认的 CSV 列表  
**输出**：`X.npy`, `y.npy`, `groups.npy`, **`train_idx.npy`**, **`test_idx.npy`**, `meta.json`

**处理流程**：

```text
按文件名匹配类别标签（跳过 *_no.csv）
  → 每类 CSV 数量对齐（取 min，如 front 1 个则 back 也用 1 个）
  → 读取 CSV → 去 warmup → 8~30 Hz 带通 → 滑窗
  → 堆叠 X (N, 8, 500), y, groups
  → GroupShuffleSplit 按 group 留出 20% 为测试集
  → 保存 train_idx / test_idx
```

**步骤验收**：

- `X.shape == (N, 8, 500)`；train + test 均含全部类别
- `meta.json` 含 `train_test_split`（样本数、组数、标签分布）

---

### 步骤 3：CSP + SVM 训练 — `03_train_csp_svm.py`

**输入**：`dataset/stage{N}/`（**仅 `train_idx`**）  
**输出**：`models/stage{N}/csp.pkl`, `svm.pkl`, `pipeline.pkl`，`reports/stage{N}/cv_results.json`

1. **5 折 GroupKFold**：仅在 **80% 训练集** 内做 CV（参考）
2. 用 **全部训练集** fit 最终 `pipeline.pkl`（**测试集不参与**）
3. `cv_results.json` 注明 `cv_scope: train_set_only`

**步骤验收**：模型未见过 `test_idx`；`cv_results.json` 与 pkl 可 load。

---

### 步骤 4：离线评估 — `04_eval_offline.py`

**输入**：`test_idx` + `models/stage{N}/pipeline.pkl`  
**输出**：`reports/stage{N}/confusion_matrix.png`, `metrics.json`

**仅在 20% 留出测试集上预测**（模型训练时未见过）。

| 指标 | 说明 |
|------|------|
| `test_set_accuracy` | **主验收指标**（hold-out 测试集） |
| 混淆矩阵 / precision / recall / F1 | 基于测试集 |
| `cv_summary` | 步骤 3 训练集内 CV（参考，非最终验收） |

**Stage 验收**（以 **测试集准确率** 为准）：

| 测试集准确率 | 动作 |
|----------------|------|
| **≥ 70%** | 进入下一 Stage |
| **65% ~ 70%** | 补录 session 后重跑 |
| **< 65%** | 补录 + 检查想象策略与电极 |

## 11. 避坑要点

1. **`_no` 勿参与训练**：`front_no.csv` 等即使与某类前缀相似也会被排除。
2. **标签看文件名**：`back.csv` 与 `back1.csv` 标签均为 1（back），不要当作不同类。
3. **不要按 9 s 切 trial**：当前数据是连续纯想象，无 trial 边界。
4. **warmup 必丢**：默认去首 2 s。
5. **CV 分组**：同一 4 s epoch 内滑窗样本必须同一 fold。
6. **front 仅 1 session**：Stage 1 不平衡，优先补录前进类。
7. **左右类先重录再去 `_no`**：再进入 Stage 3/4。
8. **Stage 隔离**：各 Stage 独立 `dataset/`、`models/` 子目录。
9. **勿重复采集端滤波**：离线只做 8~30 Hz 窄带通。

## 12. 附录：样本数估算公式

```text
usable_samples = floor((duration_sec - WARMUP_SEC) * SFREQ)
n_windows      = floor((usable_samples - WIN_SIZE) / WIN_STEP) + 1
n_groups       = ceil((duration_sec - WARMUP_SEC) / EPOCH_SEC)
```

当前默认值：`WARMUP=2s`, `WIN=2s/500点`, `STEP=0.5s/125点`, `EPOCH=4s`

单 session（83 s）→ 约 **158 窗**、**20 组**  
Stage 1（3 session：front1 + back + back1）→ 约 **475 样本**、**~60 组**

补录 front 至 2+ session、每类 10 session 后可显著提升 CV 稳定性。
