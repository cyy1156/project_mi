# 模型接入配置教程

> **适用版本**：第 11 课起（`config/models.yaml` + `models/registry.py`）  
> **读者**：操作员 / 实验室管理员 / 程序员  
> **相关文档**：[项目需求分析与技术概要.md](./项目需求分析与技术概要.md) §13、[教学计划.md](./教学计划.md) 第 11 课

---

## 1. 先回答：每接一个模型，要不要改 UI？

**不用。**

只要 UI（或命令行）是从 **`ServiceManager` + `models.yaml`** 读模型列表，新模型**只改配置和 Python 插件文件**，界面会自动出现新名字。

| 要改什么 | 谁改 | 频率 |
|----------|------|------|
| `config/models.yaml` 加一段 | 管理员 / 程序员 | 每个新模型一次 |
| `models/xxx.py` 写 `predict` | 程序员 | 每个新算法一次 |
| UI / 控制面板代码 | 程序员 | **几乎不用**（除非要新控件类型） |
| `config/default.yaml` | 管理员 | 一般不用为单个模型改 |

```text
models.yaml 登记
      ↓ 启动时 load_registry()
ServiceManager.list_models() / get_model_specs()
      ↓
UI 模型列表自动刷新（重启程序后）
      ↓
用户点「启动 xxx」→ start_model("xxx")
      ↓
ModelWorker 拉 LSL → predict → 输出进日志区
```

**日志区 / 模型输出区**也是通用的：任何模型的 `predict` 返回值都会按统一规则显示（见 §6），**不必为 demo、睡眠检测各写一块 UI**。

## 2. 三类人分别做什么

| 角色 | 会不会编程 | 要做的事 |
|------|:----------:|----------|
| **操作员** | 否 | 在 UI/CLI 点「开始采集」→ 选模型 →「启动」/「停止」 |
| **管理员** | 会改 YAML | 复制 `models.example.yaml` → `models.yaml`，填名字、窗口、权重路径 |
| **程序员** | 是 | 写 `models/xxx.py`，在 yaml 里填 `模块` + `类名` 或 `入口` |

**第一次**接入新算法需要程序员；接入之后，操作员永远用**同一个按钮/命令**切换模型名。

## 3. 配置文件在哪里

```text
config/
├── default.yaml          # 采集：串口、合成板、滤波、GUI 推流（与具体模型无关）
├── models.yaml           # 模型登记表（你要复制并维护的文件）
└── models.example.yaml   # 示例模板（可复制，不要直接改 example）
```

| 文件 | 管什么 |
|------|--------|
| `default.yaml` | 板卡、采样率、LSL、是否推 UDP 给 OpenBCI GUI |
| `models.yaml` | 有哪些模型、窗口多长、Python 类在哪、权重路径 |

改 `models.yaml` 后必须 **退出并重新打开** 控制面板/UI，列表才会更新。

## 4. `models.yaml` 字段说明

顶层 **键名** = 操作员看到的名字（如 `demo`、`睡眠检测`），区分大小写。

| 字段 | 必填 | 含义 | 示例 |
|------|:----:|------|------|
| （顶层键名） | ✓ | UI/CLI 里显示的名字 | `demo` |
| `说明` | | 给人看的备注，程序可忽略 | `打印均值方差` |
| `窗口采样点数` | ✓ | 每次 predict 用多少点；250≈1 秒 @250Hz | `250` |
| `步长采样点数` | | 滑动步长；默认窗口一半（50% 重叠） | `125` |
| `模块` | ✓ | Python 模块路径 | `models.demo_stats` |
| `类名` | * | 类插件时填 | `DemoStatsModel` |
| `入口` | * | 函数插件时填 | `predict` |
| `权重文件` | | `.pt` / `.onnx` 等，无权重可省略 | `checkpoints/sleep.pt` |
| `输入格式` | | 默认 `channels_samples`；特殊模型再改 | `channels_samples` |

\* **`类名` 与 `入口` 二选一**，不能都空。

### YAML 书写注意

- 缩进用 **2 个空格**，不要用 Tab  
- 冒号后要有空格：`说明: xxx`  
- 中文键名与英文键名等价（程序两种都认），建议与 example 保持一致

## 5. 接入新模型：完整流程

### 情况 A：只用内置 demo（零代码）

1. 复制 `config/models.example.yaml` → `config/models.yaml`  
2. 保留 `demo:` 段即可  
3. 启动控制面板 → `model list` 或 UI 模型列表应出现 `demo`  

### 情况 B：新增一个类插件（最常见）

**程序员：**

1. 在 `models/` 下新建 `my_model.py`  
2. 继承 `ModelPlugin`，实现 `load()` 和 `predict(data)`  
3. `data` 形状固定为 `(n_channels, n_samples)`，单位 **uV**（LSL 已预处理）

**管理员（或程序员）：** 在 `models.yaml` 增加：

```yaml
我的模型:
  说明: 实验用 XX 分类
  窗口采样点数: 250
  步长采样点数: 125
  模块: models.my_model
  类名: MyModel
  权重文件: checkpoints/my_model.pt   # 可选
```

**操作员：**

1. 重启 UI/控制面板  
2. `start` 采集 → 在列表里对「我的模型」点启动  
3. 在日志/模型输出区看结果  

### 情况 C：函数插件（只有 `predict` 函数）

`models/mi_model.py` 示例结构：

```python
def load():
    ...  # 可选：加载权重

def predict(data):
    ...  # data: (8, window)
    return {"label": "left", "score": 0.92}
```

`models.yaml`：

```yaml
运动想象:
  说明: MI 二分类
  窗口采样点数: 500
  步长采样点数: 250
  模块: models.mi_model
  入口: predict
  权重文件: checkpoints/mi.onnx
```

## 6. 模型输出与 UI 日志（不用改界面）

`ModelWorker` 每次 `predict` 后会输出一行，规则是通用的：

| `predict` 返回值 | 日志/UI 显示方式 |
|------------------|------------------|
| `dict`，且含 `mean_uv` / `std_uv` | 格式化为 `mean=... uV  std=... uV`（demo 同款） |
| 其它 `dict` | 整段 dict 转字符串显示 |
| 字符串 / 数字等 | 直接 `str(result)` 显示 |

**建议约定（方便人读，UI 不用改）：**

```python
return {
    "label": "wake",      # 分类结果
    "score": 0.87,        # 置信度
    "mean_uv": 12.3,      # 若有，会走 demo 同款格式
}
```

以后做 UI 时，日志区只需：

- 订阅**统一日志队列**（系统日志 + 模型输出）  
- 每行前缀 `[模型/名字]`  
- **不按模型名写 if/else**

若某模型需要图表（频谱、混淆矩阵），那是 **Phase 2 可选功能**，仍可通过 `predict` 返回结构化 dict，由**一个**「结果渲染器」按 key 解析，而不是每个模型改 UI。

## 7. 窗口与采样率

| 项目 | 约定 |
|------|------|
| 采样率 | 与 Cyton 一致，默认 **250 Hz**（在 `default.yaml`） |
| 窗口 | 在 `models.yaml` 的 `窗口采样点数` 填，如 250=1s，500=2s |
| 步长 | `步长采样点数`；不填则程序用 **窗口的一半** |
| 通道数 | 默认 **8**，与 `default.yaml` 的 `通道数` 一致 |

改窗口后：**改 yaml → 重启程序 → 再启动模型**，无需改 UI。

---

## 8. 权重文件路径

`权重文件` 可以是：

- 相对项目根目录：`checkpoints/sleep.pt`  
- 绝对路径：`D:/data/models/sleep.pt`  

程序会先找相对路径，再找 `项目根/权重文件`。  
权重在插件的 `load()` 里读取（类插件）或模块的 `load()`（函数插件）。

## 9. 操作员速查

```text
1. 打开 UI / eeg_control_panel.py
2. 开始采集（status = RUNNING）
3. 在模型列表选名字 → 启动
```

## 11. 常见问题

| 问题 | 原因 | 处理 |
|------|------|------|
| 列表里没有新模型 | yaml 未保存 / 未重启 | 检查 `config/models.yaml`，quit 重开 |
| `未登记模型` | 名字拼错 | 从 list 复制，区分大小写 |
| `未找到 LSL 流` | 未 start 采集 | 先 RUNNING 再启模型 |
| `import` 失败 | `模块` 路径错 | 应对 `models/xxx.py` → `models.xxx` |
| 缺少类名/入口 | yaml 漏填 | `类名` 或 `入口` 二选一 |
| 改了窗口不生效 | 未重启 | quit 后重开 |
| 示例 `睡眠检测` 报错 | 只有 yaml 没有 `.py` | 删掉示例段或让程序员补文件 |

## 12. 与 UI 设计的对应关系（给你做界面时参考）

| UI 区域 | 数据来源 | 是否随模型变 |
|---------|----------|:------------:|
| 模型列表 | `get_model_specs()` | 自动 |
| 启动/停止按钮 | `start_model(name)` / `stop_model(name)` | 自动 |
| 运行中标记 | `get_running_models()` | 自动 |
| 系统日志 | 采集、连接、错误 | 通用 |
| 模型输出 | ModelWorker → 日志队列 | 通用格式 |
| GUI 连接说明 | 静态帮助文本 | 与模型无关 |

**结论：接入 N 个模型 = 维护 N 段 yaml + N 个 `models/*.py`，UI 只写一次。**

## 13. 最小示例：从零加一个叫 `hello` 的模型

**①** `models/hello.py`：

```python
from __future__ import annotations
import numpy as np
from models.base import ModelPlugin

class HelloModel(ModelPlugin):
    name = "hello"
    window_size = 250
    hop_size = 125

    def load(self) -> None:
        pass

    def predict(self, data: np.ndarray) -> dict:
        return {"msg": "ok", "mean_uv": float(np.mean(data))}
```

**②** `config/models.yaml` 追加：

```yaml
hello:
  说明: 最小示例插件
  窗口采样点数: 250
  步长采样点数: 125
  模块: models.hello
  类名: HelloModel
```

**③** 重启 → `start` → `model start hello` → 日志应出现 `[模型/hello]`。
