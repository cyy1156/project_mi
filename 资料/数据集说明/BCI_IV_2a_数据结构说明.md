本文档描述 `DATA/archive/` 目录下 `.mat` 文件的数据组织方式，基于对本项目实际文件的解析结果编写。

## 2. `.mat` 文件是什么？

`.mat` 是 MATLAB 的二进制数据格式。用 Python 的 `scipy.io.loadmat` 读取后，每个文件在顶层只暴露一个核心变量：

```
.mat 文件
└── data    # 类型: numpy.ndarray, dtype=object
```

`data` 是一个**对象数组**，每个元素对应实验中的一个 **run（实验段）**，内部为 MATLAB struct，包含脑电信号及元数据。

---

## 3. 整体层级结构

可以用下面的层级来理解数据：

```
A01T.mat
└── data                          # 9 个 run（A04 为 7 个）
    ├── data[0]                   # run 0：校准段，通常无标签
    ├── data[1]                   # run 1：校准段，通常无标签
    ├── data[2]                   # run 2：校准段，通常无标签
    ├── data[3]                   # run 3：正式 MI，48 trials
    ├── data[4]                   # run 4：正式 MI，48 trials
    ├── data[5]                   # run 5：正式 MI，48 trials
    ├── data[6]                   # run 6：正式 MI，48 trials
    ├── data[7]                   # run 7：正式 MI，48 trials
    └── data[8]                   # run 8：正式 MI，48 trials
```

### 3.1 两类 run

| 类型 | 典型索引（A01~A03, A05~A09） | 是否含 `trial` / `y` | 用途 |
|------|------------------------------|----------------------|------|
| 校准 run | 0, 1, 2 | 否（为空） | 设备校准，无分类标签 |
| 正式 MI run | 3, 4, 5, 6, 7, 8 | 是（各 48 trials） | 带标签的运动想象数据 |

每个带标签受试者共有 **6 × 48 = 288** 个 trial。

### 3.2 特殊受试者 A04

`A04T.mat` 的结构与其他受试者不同：

| 项目 | A01~A03, A05~A09 | A04 |
|------|------------------|-----|
| run 总数 | 9 | **7** |
| 校准 run | 0, 1, 2 | 仅 **0** |
| 带标签 run | 3, 4, 5, 6, 7, 8 | **1, 2, 3, 4, 5, 6** |
| trial 总数 | 288 | 288 |

处理 A04 时，不能硬编码「只取 run 3~8」，应改为：**选取 `trial` 非空的 run**。

## 4. 单个 run 的字段说明

每个 `data[i]` 包含以下字段（MATLAB struct 成员）：

| 字段 | 类型 | 形状/长度 | 说明 |
|------|------|-----------|------|
| `X` | `float` 二维数组 | `(n_samples, 25)` | 连续脑电信号，单位通常为 μV |
| `trial` | `int` 一维数组 | `(n_trials,)` | 每个 trial 的 cue 出现时刻（采样点索引，从 0 开始） |
| `y` | `int` 一维数组 | `(n_trials,)` | 每个 trial 的类别标签（1~4） |
| `fs` | `int` 标量 | — | 采样率，固定为 **250 Hz** |
| `classes` | 字符串数组 | 长度 4 | 类别名称列表 |
| `artifacts` | `int` 一维数组 | `(n_trials,)` | 坏试次标记：`0` = 正常，`1` = 含伪迹 |
| `gender` | 字符串 | — | 受试者性别（`male` / `female`） |
| `age` | `int` 标量 | — | 受试者年龄 |

### 4.1 `X` — 连续脑电矩阵

`X` 是数据的**主体**，按时间连续存储，而不是按 trial 分行。

```
X[s, c] = 第 s 个采样点、第 c 个通道的电压值

s ∈ [0, n_samples - 1]
c ∈ [0, 24]
```

**示例（A01, run 3）：**

```
shape: (96735, 25)
含义: 96735 个时间点 × 25 个通道
时长: 96735 / 250 ≈ 386.9 秒
```

前几行示意：

| 时间点 | 通道0 | 通道1 | 通道2 | 通道3 | 通道4 |
|--------|-------|-------|-------|-------|-------|
| 0 | 0.34 | 0.24 | -3.22 | -7.86 | -6.15 |
| 1 | -6.35 | -7.96 | -10.50 | -15.33 | -9.18 |
| 2 | -1.81 | -7.18 | -8.15 | -10.64 | -4.79 |

### 4.2 `trial` — 试次起始位置

`trial[k]` 表示第 `k` 个 trial 的 **cue（提示）出现时刻**在 `X` 中的行索引（采样点）。

**示例（A01, run 3）：**

```
trial = [251, 2254, 4172, 6124, 8132, ...]   # 共 48 个
fs    = 250
```

| trial_id | 采样点 | 时间（秒） | 计算方式 |
|----------|--------|------------|----------|
| 0 | 251 | 1.004 | 251 / 250 |
| 1 | 2254 | 9.016 | 2254 / 250 |
| 2 | 4172 | 16.688 | 4172 / 250 |

相邻 trial 间隔约 **8.0 ± 0.3 秒**（A01 实测均值 8.04 s）。

### 4.3 `y` — 类别标签

`y[k]` 与 `trial[k]` 一一对应，取值为 1~4：

| y 值 | 类别名（`classes`） | 任务含义 |
|------|---------------------|----------|
| 1 | left hand | 想象左手运动 |
| 2 | right hand | 想象右手运动 |
| 3 | feet | 想象双脚运动 |
| 4 | tongue | 想象舌头运动 |

**示例（A01, run 3 前 5 个 trial）：**

```
trial = [251,  2254, 4172, 6124, 8132]
y     = [  4,     3,    2,    1,    1]
含义  = [舌头,  双脚,  右手,  左手,  左手]
```

每个带标签 run 内 4 类各 12 个 trial，均衡分布。

### 4.4 `artifacts` — 坏试次标记

| 值 | 含义 | 处理建议 |
|----|------|----------|
| 0 | 正常试次 | 保留 |
| 1 | 含眼电/肌电等伪迹 | 分析或训练时建议剔除 |

### 4.5 元数据字段

| 字段 | 说明 |
|------|------|
| `fs` | 采样率，本数据集恒为 250 |
| `classes` | `['left hand', 'right hand', 'feet', 'tongue']` |
| `gender` / `age` | 受试者人口统计学信息，在同一文件各 run 间一致 |

## 6. 实验范式与 trial 时间结构

BCI Competition IV 2a 的单次 trial 时间结构（相对 cue 时刻）：

```
      固定十字          cue 提示           运动想象              休息
  |←—— 2 s ——→|←— 1.25 s —→|←———— 4 s ————→|←—— ~0.5 s ——→|
  |           |            |                |                |
 -2s          0s          1.25s            4s              ~8s（下一 trial）
```

| 阶段 | 时间（相对 cue） | 说明 |
|------|------------------|------|
| 固定 | -2 ~ 0 s | 注视十字 |
| 提示 | 0 ~ 1.25 s | 屏幕箭头指示想象类别 |
| 运动想象 | 0 ~ 4 s | 执行想象任务（分析最常用窗口） |
| 试次间隔 | ~8 s | 两次 cue 之间约 8 秒 |

### 6.1 常用 epoch 切分参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `tmin` | -0.5 s | epoch 起点（相对 cue） |
| `tmax` | 4.0 s | epoch 终点 |
| `baseline` | (-0.5, 0) s | 基线校正窗口 |
| 滤波频段 | 8~30 Hz | μ + β 节律，MI 分析常用 |

## 7. trial 与连续信号的关系

这是理解本数据集最关键的部分：

```
连续信号 X（一条长波形）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
         ↑              ↑              ↑
      trial[0]       trial[1]       trial[2]
      y[0]=4         y[1]=3         y[2]=2
      舌头           双脚           右手
```

- `X` 本身**不包含** trial 边界，只有 `trial` 数组标定了切分位置
- 获取某个 trial 的脑电片段：`X[trial[k] + start : trial[k] + end, :]`
- 其中 `start`、`end` 为相对 cue 的采样点偏移，例如 `-0.5 s` → `start = -int(0.5 * fs)`

**示例：提取 A01 run 3 trial 0，cue 后 0.5 s、前 5 个 EEG 通道**

```python
cue = 251
segment = X[cue : cue + int(0.5 * 250), :5]   # shape: (125, 5)
```

## 10. 等价的「表格」视图

`.mat` 原始存储不是关系型表格，但可以映射为以下逻辑表，便于导出 CSV 或命令行查看。

### 10.1 Run 概览表（summary）

一行 = 一个 run。

| subject | run_id | n_samples | n_channels | fs | n_trials | n_artifacts | has_labels |
|---------|--------|-----------|------------|-----|----------|-------------|------------|
| A01 | 3 | 96735 | 25 | 250 | 48 | 2 | true |
| A01 | 4 | 96735 | 25 | 250 | 48 | 4 | true |
| ... | ... | ... | ... | ... | ... | ... | ... |

### 10.2 Trial 清单表（trials）— 最实用的表格化形式

一行 = 一个 trial。

| subject | run_id | trial_id | cue_sample | cue_time_sec | label | class_name | artifact |
|---------|--------|----------|------------|--------------|-------|------------|----------|
| A01 | 3 | 0 | 251 | 1.004 | 4 | tongue | 0 |
| A01 | 3 | 1 | 2254 | 9.016 | 3 | feet | 0 |
| A01 | 3 | 2 | 4172 | 16.688 | 2 | right hand | 0 |

A01 全部带标签 trial 共 **288 行**。

### 10.3 连续采样表（raw）— 原始 X 展开

一行 = 一个时间点。

| time_sec | Fz | FC3 | ... | C3 | C4 | ... | EOG3 |
|----------|-----|-----|-----|-----|-----|-----|------|
| 0.000 | 0.34 | 0.24 | ... | ... | ... | ... | ... |
| 0.004 | -6.35 | -7.96 | ... | ... | ... | ... | ... |

单 run 约 **9 万行**，全受试者体量极大，不建议直接作为默认导出格式。

### 10.4 Epoch 长表（epochs）— 切分后的 trial 数据

一行 = 一个 trial 在某个时刻、某个通道的值。

| subject | run_id | trial_id | time_sec | channel | value | label |
|---------|--------|----------|----------|---------|-------|-------|
| A01 | 3 | 0 | -0.40 | C3 | -5.71 | 4 |
| A01 | 3 | 0 | -0.36 | C3 | -6.88 | 4 |

适合可视化，但数据量较大，通常限定通道和时间窗口导出。

## 11. Python 读取示例

### 11.1 加载文件

```python
import scipy.io

mat = scipy.io.loadmat(
    "DATA/archive/A01T.mat",
    squeeze_me=True,
    struct_as_record=False,
)
data = mat["data"]          # shape: (9,) for most subjects
run = data[3]               # 取 run 3
```

### 11.2 访问各字段

```python
X = run.X                   # (n_samples, 25)
trials = run.trial          # (48,)
labels = run.y              # (48,)
fs = run.fs                 # 250
classes = run.classes       # ['left hand', 'right hand', 'feet', 'tongue']
artifacts = run.artifacts   # (48,)
```

### 11.3 遍历所有带标签 run

```python
for run_id, run in enumerate(data):
    if run.trial is None or len(run.trial) == 0:
        continue  # 跳过校准 run

    for trial_id, (cue, label, art) in enumerate(
        zip(run.trial, run.y, run.artifacts)
    ):
        if art == 1:
            continue  # 跳过坏试次

        cue_sec = cue / run.fs
        class_name = run.classes[label - 1]
        epoch = run.X[cue : cue + int(4.0 * run.fs), :22]  # cue 后 4s, 仅 EEG
```

### 11.4 依赖

```
scipy
numpy
```

进阶分析推荐额外安装 `mne`、`pandas`。

## 12. 常见注意事项

1. **不要假设所有受试者 run 索引相同** — A04 的带标签 run 为 1~6，不是 3~8。
2. **校准 run 无标签** — `trial` 和 `y` 为空数组，不可用于监督学习。
3. **务必处理 artifacts** — 尤其 A06（69 个）和 A09（51 个）坏试次较多。
4. **`y` 从 1 开始编号** — 访问 `classes` 时用 `classes[y - 1]`。
5. **`trial` 是采样点索引** — 转秒需除以 `fs`（250）。
6. **当前仅有 T 文件** — 无 Evaluation（E）集；E 集无公开标签，仅用于竞赛提交。
7. **不要直接打开原始 X 当表格** — 体积过大，应优先使用 trial 级或 epoch 级表。

---

## 13. 一句话总结

> 每个 `.mat` 文件 = 多段连续脑电录音（`X`）+ 每段上的试次书签（`trial`）+ 任务标签（`y`）+ 质量标记（`artifacts`）+ 实验元信息（`fs`、`classes` 等）。

数据的核心不是「一行一个样本」，而是「**连续时间序列 + 事件标注**」；做分析和可视化时，需要先用 `trial` 将 `X` 切分为单个试次（epoch）。
