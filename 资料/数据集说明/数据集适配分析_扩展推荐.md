# 运动想象 EEG 数据集适配分析与扩展推荐

> 对照标准：你们《数据集说明.docx》中的统一规则  
> - 仅左 / 右；脚舌等丢弃  
> - MI 窗：Cue 后 0–4 s（或等价绝对时间）  
> - 静息：无 Cue / 无 MI / 无伪迹，固定 4 s  
> - 统一形状：`(B, 1, 8, 1000)` @ 250 Hz  
> - 固定通道序：`Cz, C3, C4, CP3, FC4, FC3, CP4, CPz`

可视化对照可打开 Canvas：`MI-dataset-fit.canvas.tsx`（聊天侧栏）。

## 1. 对你已整理的五库：结论

| 数据集 | 适配 | 一句话 |
|--------|------|--------|
| **BCI IV 2a** | **首选（当前基线）** | 范式与切窗规则最贴；短板是 9 人 |
| **OpenBMI / Lee2019-MI** | **强烈推荐下一站** | 54 人、左右≈4 s、8 导齐全 |
| **Stieger 2021** | **推荐（进阶）** | 62 人且原生静息；需处理变长在线试次 |
| **BCI IV Dataset 1** | 可用 | 连续流+空闲有价值；评测常无标签、人少 |
| **BCI IV 2b** | **不适合主线** | 仅 3 导，无法满足固定 8 通道 |

### 1.1 BCI IV Dataset 2a（继续作为基线）

- 9 人，22 EEG @ 250 Hz，离线 cue。  
- 你们已规定：只留左/右，`t=2–6 s`，静息从间隔构造。  
- **优点**：与现有 `preprocess_lab` / 双头 / 五折完全对齐。  
- **缺点**：被试独立五折时 val 常只有 1–2 人，调参噪声大（昨夜实验已体现）。

### 1.2 OpenBMI / Lee2019-MI（下一优先下载）

- 54 人，62 导 @ 1000 Hz，左右手抓握想象，单试次 MI≈4 s。  
- 运动区 20 导子集已明确包含你们全部 8 导。  
- 下载：https://ww2.nemar.org/dataset/nm000338  
- **适合**：跨被试 / 跨会话正式验证；补齐 2a 人数不足。

### 1.3 Stieger 2021（大样本进阶）

- 62 人，最多 11 会话，左 / 右 / 双手 / **原生静息**。  
- 在线光标控制，反馈段最长约 6 s（变长）。  
- 下载：https://ww2.nemar.org/dataset/nm000339  
- **适合**：原生静息标签、纵向适应、大规模跨人。  
- **成本**：丢掉「双手」；反馈开始后切固定 4 s；降采样+选 8 导。

### 1.4 BCI IV Dataset 1

- 7 人，59 导，两动作类 + 空闲，偏异步连续解码。  
- 校准有标签、评测往往无标签。  
- **适合**：以后做连续 / 异步解码实验；**不适合**现在替代 2a 主线。

### 1.5 BCI IV Dataset 2b（建议搁置）

- 9 人，**仅双极 C3 / Cz / C4**。  
- 缺 `CP3, FC4, FC3, CP4, CPz` → 与当前 EEGNet 输入 `(8,1000)` 冲突。  
- 除非单独做「3 通道消融」，否则不要进统一多数据集管线。

## 2. 新推荐的合适数据集

### 2.1 Yang2025（WBCIC-SHU）— 强烈推荐

| 项 | 内容 |
|----|------|
| 文献 | Scientific Data 2025：https://doi.org/10.1038/s41597-025-04826-y |
| 数据 | Figshare Plus：https://doi.org/10.25452/figshare.plus.22671172 |
| 规模 | 62 人；其中 51 人为左/右二分类（2C） |
| 采集 | 约 59 EEG @ 1000 Hz；官方流程可到 250 Hz、MI 段 1000 点 |
| 时序 | cue 1.5 s + **MI 4.0 s** + break 2.0 s → 与你们 4 s 规则几乎同构 |
| 8 导 | 国际 10–20 运动区覆盖；落地前用通道表再核对 |

**为什么适合**：切窗形状最接近现有张量；人数够做稳定被试独立五折。

### 2.2 Dreyer2023 A/B/C — 强烈推荐

| 项 | 内容 |
|----|------|
| 文献 | Scientific Data 2023：https://doi.org/10.1038/s41597-023-02445-z |
| 数据 | Zenodo：https://doi.org/10.5281/zenodo.8089820 |
| 规模 | **87 人**（三子集同协议左/右 MI） |
| 采集 | 27 导 @ 512 Hz |
| 8 导 | **完整包含** Cz/C3/C4/CP3/FC4/FC3/CP4/CPz |
| 静息 | 协议含 resting / ITI，可按你们规则切 4 s |

**为什么适合**：目前公开左/右 MI 里**人数优势最大**，且通道硬约束直接满足。

### 2.3 Cho2017（GigaDB）— 可用备选

| 项 | 内容 |
|----|------|
| 文献 / 数据 | https://doi.org/10.5524/100295 ；NEMAR nm000245 |
| 规模 | 52 人，64 EEG @ 512 Hz，左/右 |
| 8 导 | 完整 |
| 注意 | MI 窗约 **3 s**（短于 4 s）→ 需裁/填或临时改窗长；含非任务态可助构造静息 |

### 2.4 Kumar2024 — 低优先

- 18 人 × 6 会话，左/右，纵向学习有意思。  
- 人数仍偏少；ITI 1.5 s 不利于稳定切 4 s 静息。  
- 可作为以后「跨会话适应」专题，不进第一批扩展。

## 4. 多库接入检查清单

- [ ] 通道名映射到固定 8 序；禁止按原始文件顺序堆叠  
- [ ] 采样率统一到 250 Hz  
- [ ] MI：Cue 后（或官方 MI 起点后）固定 4 s → 1000 点  
- [ ] 静息：无 Cue / 无 MI / 无伪迹，固定 4 s，不够就丢  
- [ ] 标签：`y_task`∈{0,1}，`y_three`∈{0,1,2}（左=1，右=2，静息=0）  
- [ ] 正式报数：被试独立五折 + 内层按人 val；不用 test 调参
