# 正式评估协议：被试独立五折 + 内层验证早停

> **性质**：评估与划分约定（文档先行）。  
> **约定**：本文只改文档、**不改**仓库 `.py`；现有代码仍是「全体试次混合 8:2」，与本文正式协议不同。  
> **适用**：阶段 A（`y_task`）与阶段 B（`y_three`）正式报数时都应遵守同一套划分逻辑。

---

## 0. 一句话

**按被试（人）划分，不做试次打散；外层五折定测试集；剩余人里再按人抽 20% 做验证集；用验证集 F1 早停；测试集只在最后评一次；随机种子只为复现同一次五组划分，不是用多种子代替五折。**

---

## 1. 角色分清：train / val / test

| 集合 | 谁进这个集合 | 训练时干什么 | 什么时候看指标 |
|------|--------------|--------------|----------------|
| **train** | 若干**整人**的全部试次 | 更新参数 | 可看，通常很高，**不能当最终结论** |
| **val** | 另一些**整人**（与 train/test 人不重叠） | **不**更新参数；选 best、**早停** | 每个 epoch 看；用 **F1**（头1用 F1，头2用 F1-macro）控制早停 |
| **test** | 再另一些**整人** | **全程不碰**（不调参、不早停） | **该折训练结束后只评一次** |

硬约束：

```text
同一被试的全部试次，只能落在 train / val / test 三者之一
禁止：同一个人的部分试次在 train、部分在 val/test
```

---

## 2. 单折比例（你举的 100 人例子）

把「本折的测试组」先拿开之后，剩余人再按 **8:2（按人）** 拆成 train / val：

```text
假设一共 100 人

外层（相对全体）：
  测试集 test  = 20 人     ← 约 20%（五折里的一整组）
  剩余池       = 80 人     ← 约 80%（训练+验证合计）

内层（只在这 80 人里、仍按人）：
  验证集 val   = 80 × 20% = 16 人
  训练集 train = 80 − 16  = 64 人

因此相对全体 100 人：
  train : val : test ≈ 64 : 16 : 20
```

口误澄清：

- 「训练集+验证集 : 测试集 = 8:2」→ 指的是 **人**，不是试次条数。  
- 「从训练集当中再抽 20% 验证」→ 更准确是：从 **剩余 80% 的人**里再抽 20% 的人做 val（不是从试次里随机 20%）。

---

## 3. 五折怎么划（外层）

```text
全体被试（如 100 人）
    │  用固定 seed 打乱顺序后，均分成 5 个小组 G0..G4
    │  （尽量人数接近；不能整除时允许差 1 人）
    ▼
折 0: test=G0,  剩余=G1∪G2∪G3∪G4 → 再按人 8:2 得 train/val
折 1: test=G1,  剩余=其余四组     → 再按人 8:2 得 train/val
折 2: test=G2,  …
折 3: test=G3,  …
折 4: test=G4,  …
```

每一折都：

1. 在本折 **train** 上训练（可看 train 精度，一般会很高）  
2. 每个 epoch 在本折 **val** 上看 F1，用于 **早停 / 存 best**  
3. 训练结束后，把 best 权重拿到本折 **test** 上**只测一次**  
4. 记下该折的 test 指标  

五折都跑完后，报告 **5 个 test 指标的均值 ± 标准差**（不要只报最好的一折）。

### 随机种子是干什么的

| 正确用法 | 错误用法 |
|----------|----------|
| 固定一个 `seed`，保证「100 人怎么分成 5 组」可复现 | 用 5 个不同 seed 各划一次，假装成「五折」 |
| 内层从剩余人抽 val 时，也用**同一套可复现规则**（如 `seed + fold_id`） | 每次运行重新随机抽人，导致别人无法复现 |

五折的本质是：**人已经分成 5 组，轮流当测试组各测一遍**；seed 只是让「分组结果」固定下来。

## 4. 单折训练流程（含早停）

```text
for fold in 0..4:
    按第 3 节得到 train_subjects / val_subjects / test_subjects
    按人取出对应试次 → train_loader / val_loader / test_loader

    初始化模型（阶段 B 可从该折的头1 best 迁移主干，见阶段 B 文档）
    best_score = -inf
    bad_epochs = 0
    patience = 例如 15   # 连续多少个 epoch 没有提升就停

    for epoch in 1..max_epochs:
        在 train 上更新参数
        在 val 上算 F1（头1）或 F1-macro（头2）
        可选：打印 train 指标（预期偏高，仅作参考）

        if val_F1 > best_score:
            best_score = val_F1
            保存 best 权重
            bad_epochs = 0
        else:
            bad_epochs += 1
            if bad_epochs >= patience:
                break   # 早停：以验证集 F1 为准

    加载 best 权重
    在 test 上评估一次 → 记录该折结果
    # 注意：禁止根据 test 再改超参 / 再早停
```

选模与早停**只认 val**；test 是锁箱。

### 5.2 三个划分函数分别干什么（白话）

```text
全体被试名单
    │
    ▼
make_subject_groups
    「把人固定分成 5 组 G0..G4」（只做分组，不碰试次）
    │
    ▼
每一折：取出 test 组，剩下的人叫 remain
    │
    ▼
split_train_val_subjects
    「从 remain 里再按人抽出约 20% 当 val，其余当 train」
    │
    ▼
iter_subject_kfold
    「把上面两步串起来，并生成与试次对齐的 True/False 掩码」
    得到：本折哪些试次属于 train / val / test
```

| 函数 | 输入 | 输出 | 一句话 |
|------|------|------|--------|
| `make_subject_groups` | 被试 ID 列表、折数、seed | 5 个小组（每组若干人） | 外层五折的「分班」 |
| `split_train_val_subjects` | 本折非测试的人、val 比例、seed、fold_id | `train_subjects`, `val_subjects` | 内层按人 8:2 |
| `iter_subject_kfold` | **每个试次对应的被试 ID**（与 `X` 等长） | 逐折：`masks` + 三组人名 | 训练时真正用来切数组 |

### 5.3 完整示例代码（带行级注释）

**以后粘贴位置：** `code/preprocess_lab/src/steps/split_subjects.py` 文件末尾（`split_by_subject` 后面）追加。

```python
# ========== 以下追加到 split_subjects.py（正式五折用） ==========
from __future__ import annotations
# 若文件顶部已有 import numpy，不要重复；这里仅示意完整性

import numpy as np


def make_subject_groups(
    unique_subjects: list[str],
    n_folds: int = 5,
    seed: int = 42,
) -> list[list[str]]:
    """
    作用：把全部被试打乱后，尽量均分成 n_folds 组。

    例子（10 人, n_folds=5）→
      [["A01","A02"], ["A03","A04"], ... 共 5 组]
    之后：第 k 折用第 k 组当 test。

    注意：seed 固定后，分组结果可复现；不要用 5 个 seed 冒充五折。
    """
    rng = np.random.default_rng(seed)
    # 先 sorted：避免「subjects 传入顺序不同」导致分组不同
    subs = np.array(sorted(unique_subjects))
    rng.shuffle(subs)
    # array_split：不能整除时允许组大小差 1（如 9 人 → 2,2,2,2,1）
    groups = [list(g) for g in np.array_split(subs, n_folds)]
    return groups


def split_train_val_subjects(
    remain_subjects: list[str],
    val_ratio: float = 0.2,
    seed: int = 42,
    fold_id: int = 0,
) -> tuple[list[str], list[str]]:
    """
    作用：从「本折非测试被试」里，按人抽出约 val_ratio 做验证集。

    例子：remain=80 人, val_ratio=0.2 → val≈16 人, train≈64 人。

    seed + fold_id：每折可复现，且折与折之间抽到的 val 人不完全相同。
    """
    rng = np.random.default_rng(seed + fold_id)
    remain = np.array(sorted(remain_subjects))
    rng.shuffle(remain)

    n_val = 0
    if len(remain) > 1:
        n_val = max(1, int(round(len(remain) * val_ratio)))
        n_val = min(n_val, len(remain) - 1)  # 至少留 1 人给 train

    val_subjects = remain[:n_val].tolist()
    train_subjects = remain[n_val:].tolist()
    return train_subjects, val_subjects


def iter_subject_kfold(
    subjects_per_trial: np.ndarray,
    n_folds: int = 5,
    val_ratio: float = 0.2,
    seed: int = 42,
):
    """
    作用：生成每一折的划分结果（按人独立）。

    参数 subjects_per_trial：
      形状 (N,)，与 X 第 0 维等长。
      例如第 i 条试次来自被试 "A03"，则 subjects_per_trial[i] == "A03"。
      对应文件：preprocess_lab/out/bci2a/bci2a_subjects.npy

    每一折 yield 一个 dict：
      fold            : 0..4
      train_subjects  : 本折训练用的人名列表
      val_subjects    : 本折验证用的人名列表
      test_subjects   : 本折测试用的人名列表
      masks           : {"train"|"val"|"test" -> 长度为 N 的 bool 数组}
                        True 表示该试次属于该集合

    训练时用法：
      X_tr = X[masks["train"]]
      y_tr = y[masks["train"]]
      ...
    """
    unique = sorted(set(subjects_per_trial.tolist()))
    groups = make_subject_groups(unique, n_folds=n_folds, seed=seed)

    for fold_id, test_subjects in enumerate(groups):
        # 除了本折 test 组以外的所有人
        remain = [s for i, g in enumerate(groups) if i != fold_id for s in g]
        train_subjects, val_subjects = split_train_val_subjects(
            remain, val_ratio=val_ratio, seed=seed, fold_id=fold_id
        )

        subj = np.asarray(subjects_per_trial)
        masks = {
            "train": np.isin(subj, train_subjects),
            "val": np.isin(subj, val_subjects),
            "test": np.isin(subj, test_subjects),
        }

        # 人不能重叠
        assert not set(train_subjects) & set(val_subjects)
        assert not set(train_subjects) & set(test_subjects)
        assert not set(val_subjects) & set(test_subjects)

        yield {
            "fold": fold_id,
            "train_subjects": train_subjects,
            "val_subjects": val_subjects,
            "test_subjects": list(test_subjects),
            "masks": masks,
        }
```

```text
五折下来：每个人恰好当且仅当一次 test 组成员
```

```python
# 追加到 dataset.py

class ArrayTaskDataset(Dataset):
    """五折用：已经按 mask 切好的 X, y_task。"""

    def __init__(self, X: np.ndarray, y: np.ndarray):
        X = np.asarray(X, dtype=np.float32)
        if X.ndim == 4 and X.shape[1] == 1:
            X = X[:, 0, :, :]
        assert X.ndim == 3 and X.shape[1:] == (8, 1000), X.shape
        self.X = X
        self.y = np.asarray(y, dtype=np.int64)
        assert len(self.X) == len(self.y)

    def __len__(self) -> int:
        return len(self.X)

    def __getitem__(self, idx: int):
        return torch.from_numpy(self.X[idx]), torch.tensor(self.y[idx], dtype=torch.long)


class ArrayThreeDataset(Dataset):
    """五折用：已经按 mask 切好的 X, y_three。"""

        if X.ndim == 4 and X.shape[1] == 1:

        assert X.ndim == 3 and X.shape[1:] == (8, 1000), X.shape
        self.X = X
        self.y = np.asarray(y, dtype=np.int64)
        assert len(self.X) == len(self.y)

        return len(self.X)
```

## 6. 和「现在工程里在跑的」有什么区别

| | **当前工程（过渡）** | **本文正式协议（目标）** |
|--|----------------------|---------------------------|
| 划分单位 | **试次**混合 8:2 | **被试**五折 + 内层按人 8:2 |
| 同一个人 | 可同时出现在 train 与 val | **禁止**跨 split |
| 测试集 | 基本没有独立 test（val 兼评估） | 每折有独立 test，最后才测 |
| 早停 | 现实现多为「跑满 epoch + 按 val 存 best」 | **明确**用 val F1 早停 |
| 五折 | 未做 | **至少** 5 折，报 mean±std |
| seed | 用于试次 `train_test_split` | 用于**固定五组被试划分**的复现 |

因此：你现在看到的头1 / 头2 数字，应标注为 **pooled trial split 基线**，**不能**写成「被试独立五折结果」。

## 7. BCI IV 2a（约 9 人）时的人数提醒

正式协议不因人少而改变，但人数会很难看：

```text
9 人 ÷ 5 折 → 组大小大约 2,2,2,2,1
剩余约 7 人 × 20% val → val 大约 1～2 人，train 大约 5～6 人
```

含义：

- 仍然按本文做五折（每人轮流进过 test 组）。  
- val 人极少时，早停会噪；可把 `patience` 略加大，或记录「该折 val 人数」。  
- 人多起来（接近你举的 100 人）后，64/16/20 才稳定。

