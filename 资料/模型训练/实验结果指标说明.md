# 实验结果指标说明（阶段 A：静息 / 任务）

> **适用**：braindecode EEGNet 二分类基线（`y_task`：0=静息，1=任务）。  
> **代码对应**：`code/train_lab/src/metrics.py`（或训练文档中的同名示例）。  
> **目的**：看懂训练日志里打印的每个数是什么意思、怎么算、怎么用来判断模型好坏。

---

## 1. 先统一「正类 / 负类」

本项目阶段 A 约定：

| 名称 | 标签 `y_task` | 含义 |
|------|---------------|------|
| **负类（Negative）** | `0` | 静息态 |
| **正类（Positive）** | `1` | 任务态（左手或右手想象都算任务） |

后面所有「召回率、特异性、精确率」都按这个约定。  
**换正负类定义，同一组预测算出来的数字会变**，所以报告里必须写清：正类=任务。

---

## 2. 混淆矩阵：所有指标的共同基础

模型对每个试次给出预测：`0` 或 `1`。和真实标签对比，只能落入下面四格之一：

|  | **预测 = 任务 (1)** | **预测 = 静息 (0)** |
|--|----------------------|---------------------|
| **真实 = 任务 (1)** | **TP** 真正例 | **FN** 假负例 |
| **真实 = 静息 (0)** | **FP** 假正例 | **TN** 真负例 |

### 四个格子的人话解释

| 符号 | 英文全称 | 中文 | 发生了什么 |
|------|----------|------|------------|
| **TP** | True Positive | 真正例 | 本来是任务，模型也说任务 → **判对** |
| **TN** | True Negative | 真负例 | 本来是静息，模型也说静息 → **判对** |
| **FP** | False Positive | 假正例 | 本来是静息，模型却说任务 → **误报**（虚惊） |
| **FN** | False Negative | 假负例 | 本来是任务，模型却说静息 → **漏检**（漏掉任务） |

记忆口诀：

- **False Positive**：假的「阳性」→ 把静息错当成任务  
- **False Negative**：假的「阴性」→ 把任务错当成静息  

代码里常见写法：

```python
cm = confusion_matrix(y_true, y_pred, labels=[0, 1])
# cm = [[TN, FP],
#       [FN, TP]]
tn, fp, fn, tp = cm.ravel()  # 摊成一维后按顺序取出四个数
```

**总数关系：**

```text
总样本数 N = TP + TN + FP + FN
真实任务数 = TP + FN
真实静息数 = TN + FP
预测为任务数 = TP + FP
预测为静息数 = TN + FN
```

---

## 3. 各指标详解

下面公式里的 TP/TN/FP/FN 都来自上一节混淆矩阵。

---

### 3.1 Accuracy（分类准确率）

**公式：**

\[
\mathrm{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}
\]

**含义：** 全部样本里，预测正确的比例。

**优点：** 直观、好懂。  
**缺点：** 类别不均衡时会「骗人」。  
例如 90% 都是任务、10% 是静息，模型永远猜「任务」，准确率也能到约 90%，但几乎不会识别静息。

**本项目建议：** 一定要看，但**不要只看** Accuracy。

---

### 3.2 Recall / Sensitivity / TPR（召回率 / 灵敏度）

**公式：**

\[
\mathrm{Recall} = \frac{TP}{TP + FN}
\]

**含义：** 在所有**真实为任务**的样本里，被模型正确找出的比例。  
关心的是：**任务别漏太多**。

| Recall 高 | Recall 低 |
|-----------|-----------|
| 任务大多能被检出 | 很多任务被当成静息（FN 多） |

别名：

- **Sensitivity（灵敏度）**  
- **TPR（True Positive Rate，真阳性率）**

与本项目的关系：若你更怕「有运动想象却没检测出来」，就重点盯 Recall。

---

### 3.3 Specificity / TNR（特异性）

**公式：**

\[
\mathrm{Specificity} = \frac{TN}{TN + FP}
\]

**含义：** 在所有**真实为静息**的样本里，被模型正确判成静息的比例。  
关心的是：**别把静息乱报成任务**。

| Specificity 高 | Specificity 低 |
|----------------|----------------|
| 静息大多判对 | 很多静息被当成任务（FP 多，误报多） |

别名：**TNR（True Negative Rate，真阴性率）**。

与本项目的关系：BCI 里若静息被频繁误触发「进入任务」，体验会很差，Specificity 很重要。

---

### 3.4 Precision / PPV（精确率）

**公式：**

\[
\mathrm{Precision} = \frac{TP}{TP + FP}
\]

**含义：** 模型**说成任务**的那些样本里，真正是任务的比例。  
关心的是：**报「任务」时有多靠谱**。

| Precision 高 | Precision 低 |
|--------------|--------------|
| 报任务比较准 | 报任务里掺了很多静息（误报） |

别名：**PPV（Positive Predictive Value）**。

和 Recall 的对比（很重要）：

| | 从谁的角度看 |
|--|----------------|
| Recall | 从「真实任务」出发：有没有漏掉 |
| Precision | 从「模型喊任务」出发：喊得准不准 |

可以一个很高、另一个很低，所以常一起看，或看 F1。

---

### 3.5 F1-score（F1 分数）

**公式：**

\[
\mathrm{F1} = \frac{2 \cdot \mathrm{Precision} \cdot \mathrm{Recall}}{\mathrm{Precision} + \mathrm{Recall}}
\]

**含义：** Precision 与 Recall 的**调和平均**。两者都高时 F1 才高；有一个很低会把 F1 拉下来。

本项目训练脚本里常用 **验证集 F1** 选 `best_task.pt`，就是为了避免只优化准确率或只优化某一个方向。

---

### 3.6 Balanced Accuracy（平衡准确率）

**公式：**

\[
\mathrm{Balanced\ Accuracy} = \frac{\mathrm{Recall} + \mathrm{Specificity}}{2}
\]

**含义：** 对「任务类表现」和「静息类表现」各给一半权重再平均。  
类别数量差很多时，往往比普通 Accuracy **更公平**。

---

### 3.7（补充）你可能在其他实验里见到的指标

阶段 A 二分类日志里不一定打印，但读论文 / 对照 `Desktop/model` 时常见：

| 指标 | 含义（简要） |
|------|----------------|
| **Kappa（Cohen's κ）** | 扣除「碰巧猜对」后的一致性；多分类基线常用 |
| **macro-F1** | 每一类先算 F1 再平均；四分类时常用 |
| **ROC-AUC** | 不同阈值下区分正负类的能力；需用概率分数 |
| **Confusion Matrix 本身** | 不只看一个数，直接看四格/多格分布 |

阶段 A 优先掌握：Accuracy、Recall、Specificity、Precision、F1。

## 4. 指标之间怎么联动（读日志用）

假设验证集共 100 个试次，混淆矩阵为：

```text
TP=40, FN=10, TN=35, FP=15
```

则：

| 指标 | 计算 | 结果 |
|------|------|------|
| Accuracy | (40+35)/100 | 0.75 |
| Recall | 40/(40+10) | 0.80 |
| Specificity | 35/(35+15) | 0.70 |
| Precision | 40/(40+15) | ≈0.73 |
| F1 | 2×0.73×0.80/(0.73+0.80) | ≈0.76 |
| Balanced Acc | (0.80+0.70)/2 | 0.75 |

**读法示例：**

- Recall=0.80：真实任务里约 80% 被找到，漏了 20%。  
- Specificity=0.70：真实静息里约 70% 判对，30% 被误报成任务。  
- Precision≈0.73：模型喊「任务」时，大约 73% 是真任务。

## 5. 常见「好看但有问题」的情况

| 现象 | 可能含义 | 该信什么 |
|------|----------|----------|
| Accuracy 很高，Specificity 很低 | 几乎总猜任务 | 模型偏科，不能只报准确率 |
| Recall≈1，Precision 很低 | 几乎从不说静息 | 漏检少但误报多 |
| Precision≈1，Recall 很低 | 很少敢报任务 | 报得准但漏很多任务 |
| train 全很高，val 很差 | 过拟合 | 以 **val** 指标为准 |
| 各项都在 0.5 附近 | 接近瞎猜（二分类） | 查数据、标签、输入形状 |

**二分类随机基线（类大致均衡时）：** Accuracy ≈ **0.5**。  
明显长期低于或贴着 0.5，说明几乎没学到有效信息。

## 6. 和本项目日志的对应关系

训练时类似输出：

```text
===== [val] 分类头1（静息=0 / 任务=1）braindecode EEGNet =====
  混淆矩阵: TP=.. TN=.. FP=.. FN=..
  Accuracy      分类准确率   = ...
  Recall        召回率/灵敏度 = ...
  Specificity   特异性       = ...
  Precision     精确率       = ...
  F1-score      F1          = ...
  Balanced Acc  平衡准确率   = ...
```

建议记录实验时至少记下（验证集）：

1. TP / TN / FP / FN  
2. Accuracy、Recall、Specificity、F1  
3. 训练/验证样本数（或 `bincount`）  
4. 随机种子、epoch、是否 best 权重  

写报告时建议同时给出：**准确率 + 召回率 + 特异性**（必要时加 F1），并注明正类=任务。

## 7. 阶段 B（三分类）时指标会怎样变（预习）

以后若开 `head_three`（空闲/左/右，三类）：

- 混淆矩阵变成 **3×3**，不再是简单的一个 TP/TN。  
- 常用 **Accuracy、macro-F1、每类 Recall**，或对「左 vs 右」单独做子分析。  
- 「特异性」对多类没有唯一标准定义，一般改为**按类**报告，或继续只在二分类头上用本节定义。

阶段 A 先把二分类这套指标吃透即可。

## 8. 一句话对照表

| 你想回答的问题 | 看哪个指标 |
|----------------|------------|
| 总体对了多少？ | Accuracy（小心不均衡） |
| 任务有没有漏掉？ | Recall |
| 静息会不会被误触发？ | Specificity |
| 模型喊「任务」靠不靠谱？ | Precision |
| 召回和精确要折中？ | F1 |
| 两类是否都还行？ | Balanced Accuracy |
