# 训练与数据相关名词对照（trial / epoch / session / 被试）

> **目的**：把容易混在一起的词分开，并说明它们在本项目里各指什么。  
> **现状（工程）**：BCI2A 多人试次合并后，按**试次** 8:2 得 train/val（可跨人泄漏）。  
> **正式目标（文档）**：被试独立五折 + 内层按人验证 + val F1 早停 → 见 `正式评估协议_被试独立五折.md`。

---

## 1. 一页总览

```text
数据集 BCI IV 2a
  └─ 被试 subject（如 A01）
        └─ 会话 session（如 A01T = Training；A01E = Evaluation，你暂未用）
              └─ 多个 run（一次会话里多段连续录音）
                    └─ 多个 trial 试次（每个 4s 窗 → 一条样本）
                          │
                          ├─【过渡】试次混合 8:2 → train / val
                          │
                          └─【正式】按人五折：
                                test=一组人；剩余人再按人 8:2 → train / val
                                例 100 人：64 train / 16 val / 20 test
                          │
                          └─ 训练时
                                └─ epoch 轮次：把 train 全部按 batch 过一遍
                                      └─ batch：一次喂给模型的一小撮 trial（如 32 条）
```

| 名词 | 属于哪一层 | 一句话 |
|------|------------|--------|
| **被试 subject** | 人 | 谁的数据（A01、A02…） |
| **会话 session** | 一次实验录音 | 如 `A01T` / `A01E` |
| **run** | 会话里的一段 | 连续 EEG + 若干 Cue |
| **trial 试次** | 一条训练样本 | 一个 4s 窗 + 标签 |
| **train / val** | 样本怎么用 | 训练用 / 验证用（当前按试次 8:2） |
| **batch** | 训练喂数方式 | 每步用多少条 trial |
| **epoch** | 训练循环单位 | 训练集完整过一轮 |

**最容易错的两点：**

1. **epoch ≠ session**：epoch 是「训练第几轮」；session 是「哪次采集」。  
2. **现在不是「一个 epoch = 一个被试」**：每个 epoch 都重复用同一份 train 的 164 条。

---

## 2. 数据采集侧（从大到小）

### 2.1 被试（subject）

- 含义：实验参与者，如 `A01` … `A09`。  
- 本项目标签里可有 `subjects.npy`，记录每条 trial 来自谁。  
- **当前**：只有 `A01`。

### 2.2 会话（session）

- 含义：同一被试在某一天/某一阶段录的一整次数据。  
- BCI IV 2a 常见：  
  - `A0xT`：Training 会话  
  - `A0xE`：Evaluation 会话  
- **当前**：只用 `A01T.mat`，**还没用** Evaluation。  
- 会话 ≠ 训练轮次。

### 2.3 Run

- 含义：一次会话里的一段连续记录（2a 的 Training 里通常有多个 run）。  
- 预处理时：对每个 run 做滤波、切窗，再把各 run 的 trial **拼在一起**。  
- 用户一般不需要按 run 划分训练；知道「一人一会话由多 run 组成」即可。

### 2.4 试次（trial）

- 含义：模型看到的**一条样本**。  
- 本项目中：分类窗 4s @ 250Hz → 张量 `(1, 8, 1000)`，带：  
  - `y_task`：0=静息，1=任务  
  - `y_three`：0=空闲，1=左，2=右（阶段 A 训练不用）  
- 静息 trial 也是试次（由 Cue 前窗口构造），不是「非 trial」。

## 3. 划分侧（train / val / test）

### 3.1 过渡做法：按试次混合划分（当前代码）

多人试次合并后：

```text
全部 trial
    │  stratified 约 8:2（按试次，不是按人）
    ├─ train
    └─ val
```

- 同一被试的试次**可以**同时进 train 和 val。  
- 目的：先跑通离线基线；**不是**跨被试泛化实验。

### 3.2 正式做法：被试独立五折（文档已定）

```text
全体被试 → seed 复现地均分 5 组
每折: test=一组（≈20%人）
      剩余人 → 再按人抽 20% = val，其余 = train
      （100 人例：64 / 16 / 20）
早停: 只看 val 的 F1
终评: 该折 test 只测一次；五折 mean±std
```

- **随机种子**：只为复现「怎么分成五组」，**不是**用多种子代替五折。  
- 全文：`资料/模型训练/正式评估协议_被试独立五折.md`

### 3.3 其它划分（对比用）

| 方式 | 含义 | 何时用 |
|------|------|--------|
| 试次 8:2（过渡） | 同一池子随机/分层拆 train/val | 工程基线 |
| 被试独立五折（正式） | 5 组人轮流当 test + 内层 val | 正式报数 / 论文 |
| 按被试 LOSO | 某人整人做测试，其他人训练 | 人很少时的极限留一法 |
| 按会话 T→E | T 训练，E 测试 | 对齐 BCI 2a 官方评测习惯 |

### 3.4 train / val / test 在协议里的分工

| 名称 | 过渡工程 | 正式协议 | 作用 |
|------|----------|----------|------|
| **train** | 有 | 有 | 更新参数；精度常偏高，只作参考 |
| **val** | 有（常兼终评） | 有 | **不**更新参数；选 best、**早停（看 F1）** |
| **test** | 基本无 | 每折有 | 锁箱；折结束**只评一次** |

口语里说「每个 epoch 的测试」→ 应指 **val**，不是 test。

## 4. 训练侧（batch / epoch）

### 4.1 Batch（批次）

- 含义：一次前向/反传用的 trial 个数。  
- 当前：`batch_size=32`（训练）、验证可用 64。  
- 例：164 条 train → 每 epoch 约 `ceil(164/32) = 6` 个 batch。

### 4.2 Epoch（轮）

- 含义：训练集中**所有 train trial** 按 batch 过完一遍，称为 1 个 epoch。  
- 当前：`epochs=50` → 同样的 164 条大约重复学习 50 遍（每遍顺序可因 `shuffle=True` 不同）。  
- **每个 epoch 的 train trial 数不变**（仍是 164）；val 仍是 42。

### 4.3 一个 epoch 里实际发生什么

```text
epoch k:
  1) 用 train 的 164 条（分多个 batch）更新 EEGNet
  2) 用 val 的 42 条算 loss 与指标（不反传）
  3) 若 val F1 更好 → 保存 best_task.pt
```

| 项目 | 数量 / 内容 |
|------|-------------|
| 被试 | 1（A01） |
| 会话 | 1（A01T） |
| 全部 trial | ≈ 206 |
| train trial / epoch | **164**（每轮都是这些） |
| val trial / epoch | **42**（每轮都是这些） |

| 约每 epoch 训练步数 | ≈ 6 |

（精确个数以 `preprocess_lab/out` 里 npy 为准；重跑预处理可能略变。）

## 6. 对照表：谁包含谁

| 较大单位 | 包含 | 不包含 |
|----------|------|--------|
| 数据集 | 多个被试 | — |
| 被试 | 一个或多个 session | 不等于一个 epoch |
| session | 多个 run、多个 trial | 不等于一轮训练 |
| train 集 | 多条 trial | 不等于「所有被试」 |
| 一个 epoch | 把 train 扫一遍 | 不等于换一个 session |

---

## 7. 常见误解

| 误解 | 正解 |
|------|------|
| 「一个 epoch 就是一个 session」 | 否。epoch 是训练轮；session 是采集会话。 |
| 「每个 epoch 换一批新 trial」 | 否。每轮仍是同一 train/val；只是权重在变。 |
| 「现在已经在做跨被试五折」 | 否。工程仍是试次混合；正式协议见 `正式评估协议_被试独立五折.md`。 |
| 「val 就是官方 Evaluation 会话」 | 否。val 是 T 会话内拆出的 20%；E 会话你还没用。 |
| 「静息不算 trial」 | 否。静息也是一条带标签的样本。 |

---

## 8. 一句话记忆

> **被试 → 会话 → run → 许多 trial；trial 再分成 train/val；训练时用多个 epoch 反复扫 train，每个 epoch 里用 batch 一小撮一小撮地喂模型。**
