## 目标与两阶段

| 阶段 | 划分方式 | 回答的问题 | 状态 |
|------|----------|------------|------|
| **过渡（当前代码）** | **全部被试的试次混在一起**，按比例分 train / val | 离线模型在「见过这些人的数据」时准不准 | ✅ 工程已实现 |

正式协议口诀（100 人例）：

```text
100 人 → 固定 seed 均分 5 组
每折 test ≈ 20 人；剩余 80 人里再按人抽 20% ≈ 16 人做 val → train ≈ 64 人
用 val F1 早停；test 终评；五折 mean±std
seed 只为复现分组，不是用多种子代替五折
```

## 12.A 全体试次混合按比例划分（当前工程）

把 A01、A02、… 所有人的试次先拼成一大份，再对**整份数据**做 80% / 20%（可分层）。  
结果只训 **一个模型**。

```text
全部试次（所有被试混在一起）
        │
        ├── 约 80% → train → 训练同一个模型
        └── 约 20% → val   → 看这个模型的准确率
```

> ⚠️ 同一个人的试次**可以**同时出现在 train 和 val。  
> 汇报时必须写清：`pooled trial split, train:val=8:2`，**不要**写成跨被试五折。

推荐按 `y_three` **分层**，避免某一折缺左/右/静。

### 参考代码

```python
import numpy as np
from sklearn.model_selection import train_test_split


def split_all_trials(
    X: np.ndarray,
    y_task: np.ndarray,
    y_three: np.ndarray,
    val_ratio: float = 0.2,
    test_ratio: float = 0.0,
    seed: int = 42,
    subjects: list[str] | np.ndarray | None = None,
) -> dict:
    """
    在全部试次上按比例划分（可来自多个被试，已拼在一起）。
    - 默认 train/val（test_ratio=0）
    - 若传入 subjects，各 split 会同步切分 subjects
    返回各键: (X, y_task, y_three) 或 (X, y_task, y_three, subjects)
    """
    assert len(X) == len(y_task) == len(y_three)
    if subjects is not None:
        subjects = np.asarray(subjects)
        assert len(subjects) == len(X)
    assert 0.0 < val_ratio < 1.0
    assert 0.0 <= test_ratio < 1.0
    assert val_ratio + test_ratio < 1.0

    idx = np.arange(len(X))
    out: dict = {}

    def _pack(ii: np.ndarray):
        if subjects is None:
            return X[ii], y_task[ii], y_three[ii]
        return X[ii], y_task[ii], y_three[ii], subjects[ii]

    if test_ratio > 0:
        idx_tv, idx_test = train_test_split(
            idx, test_size=test_ratio, random_state=seed, stratify=y_three
        )
        out["test"] = _pack(idx_test)
        val_ratio_adj = val_ratio / (1.0 - test_ratio)
        idx_train, idx_val = train_test_split(
            idx_tv,
            test_size=val_ratio_adj,
            random_state=seed,
            stratify=y_three[idx_tv],
        )
    else:
        idx_train, idx_val = train_test_split(
            idx, test_size=val_ratio, random_state=seed, stratify=y_three
        )

    out["train"] = _pack(idx_train)
    out["val"] = _pack(idx_val)
    return out
```

### 怎么理解（重要）

| 问题 | 答案 |
|------|------|
| 模型是一个人一个，还是一个总模型？ | **一个总模型** |
| 数据从哪来？ | 所有被试试次先合并，再 8:2 |
| 同一个人会不会既在 train 又在 val？ | **会**（随机抽试次时很常见） |
| 这能说明换新人也能用吗？ | **不能**；正式泛化请用 12.B / 正式评估协议 |

### 自测建议

```python
# 模拟 3 人共 100 个试次已合并
X = np.zeros((100, 1, 8, 1000), np.float32)
y_task = np.array([1] * 70 + [0] * 30)
y_three = np.array([1] * 35 + [2] * 35 + [0] * 30)
subjects = ["A01"] * 40 + ["A02"] * 30 + ["A03"] * 30

parts = split_all_trials(
    X, y_task, y_three, val_ratio=0.2, seed=42, subjects=subjects
)
n_train = parts["train"][0].shape[0]
n_val = parts["val"][0].shape[0]
print(n_train, n_val)  # 约 80 / 20
assert n_train + n_val == 100

# train/val 里都可能出现多个被试
print("train subjects:", set(parts["train"][3]))
print("val subjects:", set(parts["val"][3]))
```

### 离线准确率怎么报（过渡）

1. 合并多人 `X / y_task / y_three`（及 `subjects`）  
2. `split_all_trials(..., val_ratio=0.2)`  
3. 在 train 上训模型，在 val 上报告指标  
4. 记录：`pooled all-trial split 8:2, one model, stratify=y_three`

## 12.B 被试独立五折 + 内层验证（正式目标）

**整个人**进入 train 或 val 或 test；禁止把同一人试次打散后假装「跨被试」。

### 结构（与正式协议一致）

```text
全体被试 --(seed 复现)--> 5 个小组 G0..G4
折 k:  test = Gk
       remain = 其余人
       val   = remain 中约 20% 的人
       train = remain 中其余人
```

100 人时约 **64 train / 16 val / 20 test**；至少跑满 5 折；**随机种子只用于复现分组**。

详细说明、早停流程、完整示意代码：见  
`资料/模型训练/正式评估协议_被试独立五折.md`。

### 按已给定被试列表切一折（底层工具）

若五组与内层名单已定，可用：

```python
def split_by_subject(
    X: np.ndarray,
    y_task: np.ndarray,
    y_three: np.ndarray,
    subjects: list[str],
    test_subjects: set[str],
    val_subjects: set[str],
) -> dict:
    assert len(X) == len(y_task) == len(y_three) == len(subjects)
    assert test_subjects.isdisjoint(val_subjects)

    subj = np.asarray(subjects)
    holdout = set(test_subjects) | set(val_subjects)
    masks = {
        "train": ~np.isin(subj, list(holdout)),
        "val": np.isin(subj, list(val_subjects)),
        "test": np.isin(subj, list(test_subjects)),
    }
    return {k: (X[m], y_task[m], y_three[m]) for k, m in masks.items()}
```

五折的「生成 G0..G4 + 每折抽 val」逻辑以正式协议文档中的 `make_subject_groups` / `iter_subject_kfold` 为准（接入代码时再落到 `split_subjects.py`）。

## 两种方式别混

| | 全体混合 8:2（过渡 A） | 被试独立五折（正式 B） |
|--|------------------------|-------------------------|
| 同一人的试次 | **可以**同时进 train 和 val | **不能**分到不同 split |
| 测试集 | 常无独立 test | 每折独立 test，终评一次 |
| 早停 | 常按 val 存 best | **必须**按 val F1 早停 |
| 准确率含义 | 混合数据上的离线拟合 | 跨人泛化（五折 mean±std） |

---

## 常见坑

- 把过渡阶段 A 的准确率写成「跨被试五折」。  
- 用 5 个随机种子各划一次，当成五折（**不对**；五折是 5 组人轮流当 test）。  
- 只切了 `y_three` 忘了同步切 `y_task` / `subjects`。  
- 分层时某类过少导致 `stratify` 失败（仅过渡试次划分会遇到）。  
- 正式协议下 `val` 与 `test` 被试有交集。  
- 根据 **test** 做早停或调参（泄漏）。

## 验收清单

**试次混合（当前）：**
- [ ] `len(train)+len(val)==总试次`（无 test 时）  
- [ ] 各 split 满足 `len(X)==len(y_task)==len(y_three)`  
- [ ] 分层后三类大致都有样本  

**被试独立五折（正式，改代码后）：**
- [ ] 被试集合互斥；同一人不会出现在两个 split  
- [ ] 5 组轮流当 test；内层 val≈剩余人 20%  
- [ ] 早停看 val F1；test 只终评；报五折 mean±std
